## SubStage 2A：ButterworthIIR

**输出文件**：`src/core/filter/ButterworthIIR.js`（新建）+ `tests/unit/ButterworthIIR.test.js`

**纯函数清单**（参考 CalculateCore.cpp L288-602，并实现 v1.0.0 bug 修复）：

### 1. Biquad 系数函数（5 个）

所有函数接受显式参数，返回 `{ b0, b1, b2, a1, a2 }`，入口处先执行 Nyquist clamp。

#### makePeakBiquad(freq, q, gainDb, sampleRate)
标准 RBJ peaking filter：
```
A = 10^(gainDb / 40)
omega = 2*PI*freq/sampleRate
alpha = sin(omega) / (2*q)
b0 = 1 + alpha*A
b1 = -2*cos(omega)
b2 = 1 - alpha*A
a0 = 1 + alpha/A
a1 = -2*cos(omega)
a2 = 1 - alpha/A
归一化: b0/=a0, b1/=a0, b2/=a0, a1/=a0, a2/=a0
```

#### makeLowShelfBiquad(freq, q, gainDb, sampleRate)
```
A = 10^(gainDb/40), omega=2*PI*freq/sampleRate
beta = sqrt(A)*sin(omega)/q
aminus1=A-1, aplus1=A+1, coso=cos(omega)
a0 = aplus1 + aminus1*coso + beta
b0 = A*(aplus1 - aminus1*coso + beta)/a0
b1 = 2*A*(aminus1 - aplus1*coso)/a0
b2 = A*(aplus1 - aminus1*coso - beta)/a0
a1 = -2*(aminus1 + aplus1*coso)/a0
a2 = (aplus1 + aminus1*coso - beta)/a0
```

#### makeHighShelfBiquad(freq, q, gainDb, sampleRate)
```
A = 10^(gainDb/40), omega=2*PI*freq/sampleRate
beta = sqrt(A)*sin(omega)/q
aminus1=A-1, aplus1=A+1, coso=cos(omega)
a0 = aplus1 - aminus1*coso + beta
b0 = A*(aplus1 + aminus1*coso + beta)/a0
b1 = -2*A*(aminus1 + aplus1*coso)/a0
b2 = A*(aplus1 + aminus1*coso - beta)/a0
a1 = 2*(aminus1 - aplus1*coso)/a0
a2 = (aplus1 - aminus1*coso - beta)/a0
```

#### makeLowPassBiquad(freq, sampleRate, q = 1/Math.SQRT2)
BLT Butterworth LPF 带 Q 参数：
```
C = 1/tan(PI*freq/sampleRate)
d0 = C*C + C/q + 1
d1 = 2*(1 - C*C)
d2 = C*C - C/q + 1
b0 = 1/d0, b1 = 2/d0, b2 = 1/d0
a1 = d1/d0, a2 = d2/d0
```
当 q=1/sqrt(2) 时退化为标准 Butterworth -3dB 截止。

#### makeHighPassBiquad(freq, sampleRate, q = 1/Math.SQRT2)
BLT Butterworth HPF 带 Q 参数：
```
C = tan(PI*freq/sampleRate)
d0 = C*C + C/q + 1
d1 = 2*(1 - C*C)
d2 = C*C - C/q + 1
b0 = C*C/d0, b1 = -2*C*C/d0, b2 = C*C/d0
a1 = d1/d0, a2 = d2/d0
```
当 q=1/sqrt(2) 时退化为标准 Butterworth -3dB 截止。

### 2. 频响求值函数

#### evaluateBiquadGainDb(coeff, freq, sampleRate) → dB
标准数字滤波器频响计算：
```
w = 2*PI*freq/sampleRate
num_re = b0 + b1*cos(w) + b2*cos(2w)
num_im = -b1*sin(w) - b2*sin(2w)
den_re = 1 + a1*cos(w) + a2*cos(2w)
den_im = -a1*sin(w) - a2*sin(2w)
|H|^2 = (num_re^2 + num_im^2) / (den_re^2 + den_im^2)
return 10*log10(|H|^2)
```

#### evaluateBandGainDb(band, freq, sampleRate) → dB
根据 band.type 调用对应 make*Biquad 生成系数，再 evaluateBiquadGainDb。
若 band.bypass === true，返回 0。

### 3. 工具函数

#### qRangeFor(filterType) → { min, max }
```
PEAK:      { min: 0.4, max: 128.0 }
LOW_SHELF: { min: 0.4, max: 1.6 }
HIGH_SHELF:{ min: 0.4, max: 1.6 }
LOW_PASS:  { min: 0.4, max: 128.0 }
HIGH_PASS: { min: 0.4, max: 128.0 }
BAND_PASS: { min: 0.4, max: 128.0 }
```

#### clampFreqBelowNyquist(freq, sampleRate) → number
返回 `Math.min(freq, sampleRate / 2 * (1 - 1e-3))`

### 4. v1.0.0 bug 修复（强制实施）
- **Nyquist clamp**：每个 make*Biquad 入口先 clamp
- **Butterworth Q 解锁**：LPF/HPF 新增 q 参数
- **qmax 统一**：一律 128.0
- **不实现 BandPass**（留待 v2.x）

### 验收标准
- Peak 滤波器在 freq=center_freq 时返回 ≈ gainDb（误差 < 0.5 dB）
- LowPass 在 freq=cutoff 时返回 ≈ -3 dB（默认 q=1/sqrt(2)）
- 各 makeCoeff 在 freq > sampleRate/2 时返回值不为 NaN/Infinity
- 单元测试覆盖所有 5 种类型 + bypass + 边界条件

### 构造函数
- 调用 `createBand()` 生成 5 个默认 band
- 频率按对数均匀分布：`freq[i] = 20 * (20000/20)^(i/4)` 即 `[20, 75.7, 287.2, 1088.5, 4126.2, 20000]`... 实际上按公式 20*(20000/20)^(i/(5-1)) = [20, 75.7, 287.2, 1088.5, 4126.2, 20000] 但 etc, we can compute as: `20 * Math.pow(1000, i / 4)` where 20000/20 = 1000. So we get: i=0:20, i=1:20*1000^0.25≈20*5.623≈112.5, i=2:20*1000^0.5≈20*31.62≈632.5, i=3:20*1000^0.75≈20*177.8≈3556.6, i=4:20000

Wait, let me reconsider. The spec says "对数均匀分布" between freqMin=20 and freqMax=20000.
freq[i] = freqMin * (freqMax/freqMin)^(i/(N-1))

For N=5:
i=0: 20
i=1: 20 * (20000/20)^(1/4) = 20 * 1000^0.25 ≈ 20 * 5.623 = 112.5
i=2: 20 * 1000^0.5 ≈ 20 * 31.62 = 632.5
i=3: 20 * 1000^0.75 ≈ 20 * 177.83 = 3556.6
i=4: 20000

### 事件派发
所有 mutator 方法在状态实际变更后 dispatch CustomEvent：
| 事件名 | detail | 触发条件 |
|--------|--------|---------|
| `band-added` | `{ index, band }` | addBand 后 |
| `band-removed` | `{ index }` | removeBand 后 |
| `band-changed` | `{ index, band }` | setBandParams/setBandCount(setQRange clamp) 后 |
| `band-count-changed` | `{ newCount }` | setBandCount 后 |
| `focused-band-changed` | `{ index }` | setFocusedBandIndex 后 |
| `sample-rate-changed` | `{ rate }` | setSampleRate 后 |

| `hpf-changed` | `{ frequency, enabled }` | setHpf/setHpfFrequency/setHpfEnabled 后 |
| `model-reset` | `{}` | setBandCount 改变 band 数量时 |

事件仅在值实际变化时派发，避免无效回调。

### 验收标准
- 构造时自动添加 5 个 band
- bandAt(idx) 返回 deep clone
- 删除 band[2] 后 addBand() 返回 2（index 复用）
- setQRange(Peak, 0.4, 2.0) 将已存在的 Peak band q=5.0 clamp 到 2.0 并发出 band-changed
- 所有 setter 在值未变化时不发事件

## SubStage 2C：CurveEngine（在 2A+2B 完成后启动）

**输出文件**：`src/core/CurveEngine.js`（新建）+ `tests/unit/CurveEngine.test.js`

### 算法
1. 生成对数频率点：`freq[i] = freqMin * (freqMax/freqMin)^(i/(pointCount-1))`
2. 对每个 freq[i]，调用 `evaluateBandGainDb` 累加所有非 bypass band 的增益
3. 叠加 LPF/HPF（若 enabled）
4. 返回 `[{ freq, gainDb }]` 数组（纯频响值，不做坐标变换）

### 验收标准
- 单 Peak band gain=+6dB, freq=1kHz, q=1.0 时，总曲线在 1kHz 附近峰值 ≈ +6dB
- LPF 关闭时不影响曲线
- LPF freq=1kHz enabled 时，曲线在 1kHz 附近 ≈ -3dB
