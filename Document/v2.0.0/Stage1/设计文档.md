# Stage 1 设计文档：基础骨架与工程化

## 背景与目标

本 Stage 搭建 v2.0.0 的工程基础：创建 Vite 项目骨架、确立目录结构、定义全局类型常量和坐标映射工具。完成后后续 Stage 可以在一个干净的工程环境中增量开发。

## 当前代码状态

- 工作区仅有 `Document/` 和 `.git/`，无 `package.json`、无 `src/`、无构建配置
- 没有任何 JS 源码、测试或示例文件

## 依赖关系

- Stage 1 无前置依赖（独立于所有其他 Stage）
- 1A / 1B / 1C 三者完全并行，无相互依赖

```js
export const FilterType = Object.freeze({
  PEAK:       'peak',
  LOW_SHELF:  'lowShelf',
  HIGH_SHELF: 'highShelf',
  LOW_PASS:   'lowPass',
  HIGH_PASS:  'highPass',

  SR_96000:  96000,

  INDEX_CONFLICT:     'IndexConflict',
  INVALID_PARAMETER:  'InvalidParameter',
  NOT_INITIALIZED:    'NotInitialized',
});

export function createBand({ frequency = 1000, gain = 0, q = 1.0,
                              type = FilterType.PEAK, bypass = false } = {}) {
  return { index: -1, frequency, gain, q, type, bypass };
}

export const Defaults = Object.freeze({
  BAND_COUNT:        5,
  SAMPLE_RATE:       48000,
  GAIN_MIN:          -24,
  GAIN_MAX:          12,
  FREQ_MIN:          20,
  FREQ_MAX:          20000,
  POINT_COUNT:       500,
  LPF_FREQ_DEFAULT:  20000,
  HPF_FREQ_DEFAULT:  20,
  FREQ_TICKS:        [20, 50, 100, 200, 500, 1000, 2000, 5000, 10000, 20000],

```

要求：
- 所有常量对象用 `Object.freeze` 包裹
- `createBand()` 默认值符合上表
- 完整的 JSDoc 类型注释（`@typedef` / `@type`），确保 `tsc --emitDeclarationOnly` 可输出 `.d.ts`
- 需有对应的单元测试 `tests/unit/types.test.js`

**跨 SubStage 契约**：
- `FilterType`、`CreateBand`、`Defaults` 后续被 Stage 2A/2B/2C 和 Stage 3-7 使用
- `createBand()` 返回的 band 对象格式：`{ index: number, frequency: number, gain: number, q: number, type: string, bypass: boolean }`

**验收标准**：
- 所有常量 `Object.freeze` 包裹，不可变
- `createBand()` 空参返回 `{ index:-1, frequency:1000, gain:0, q:1.0, type:'peak', bypass:false }`
- `createBand({ frequency: 2000, gain: 3 })` 返回 `{ index:-1, frequency:2000, gain:3, q:1.0, type:'peak', bypass:false }`
- `Defaults.BAND_COUNT === 5`, `Defaults.SAMPLE_RATE === 48000`
- 单元测试覆盖：空参 / 部分参 / 全参三种 case

### SubStage 1C：CoordinateMapper

**输出文件**：`src/core/CoordinateMapper.js`（新建）

**详细规范**（来源于开发计划.md §3.3）：

```js
export class CoordinateMapper {
  constructor({ viewport, freqMin, freqMax, gainMin, gainMax }) {
    // 预计算 logFreqRatio = Math.log(freqMax / freqMin)
  }

  // 频率 → X（对数映射，clamp 到视口）
  freqToX(freqHz): number

  // X → 频率（反向）
  xToFreq(x): number

  // 增益 → Y（线性映射，Y 轴向下，gainMax 在顶部）
  gainToY(gainDb): number

  // Y → 增益（反向）
  yToGain(y): number

  // 视口管理
  setViewport({ x, y, width, height }): void
  setFreqRange(min, max): void
  setGainRange(min, max): void

  // getter
  get freqMin()
  get freqMax()
  get gainMin()
  get gainMax()
  get viewport()
}
```

**关键公式**：
- 频率对数映射：`x = vp.x + vp.w * Math.log(freq / freqMin) / logFreqRatio`
- 增益线性映射（Canvas Y 轴向下）：`y = vp.y + vp.h * (gainMax - gain) / (gainMax - gainMin)`
- 构造函数预计算 `logFreqRatio` 加速

**跨 SubStage 契约**：
- 提供给 Stage 3 (GridLayer, LabelLayer, CurveLayer, HandleLayer) 和 Stage 4 (HitTester) 使用
- viewport 格式：`{ x: number, y: number, width: number, height: number }`
- CoordinateMapper 是纯数据模块，不依赖 DOM / Canvas

**验收标准**：
- `freqToX` / `xToFreq` 互为反函数，round-trip 误差 < 1e-9
- `gainToY` / `yToGain` 互为反函数，round-trip 误差 < 1e-9
- 边界值正确：`freqToX(freqMin)` 返回 `vp.x`，`gainToY(gainMax)` 返回 `vp.y`
- `setViewport` 仅更新成员，不触发重算
- 需有对应单元测试 `tests/unit/CoordinateMapper.test.js`
- 单元测试覆盖：
  - 构造默认值
  - freqToX ↔ xToFreq round-trip 多个频率点
  - gainToY ↔ yToGain round-trip 多个增益值
  - 边界值（freqMin, freqMax, gainMin, gainMax）
  - setViewport / setFreqRange / setGainRange 后映射更新

**禁止事项**：
- 不依赖任何 DOM API（保持纯函数模块）
- 不做任何渲染逻辑
