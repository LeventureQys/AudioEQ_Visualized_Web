# v2.0.0 开发总览

> **版本**：v2.0.0
> **目标**：完整实现 AudioEQ Web Component — 基于原生 Custom Element + Canvas 2D 的音频 EQ 可视化通用组件，可在任何 HTML / React / Vue 项目中直接使用，对外暴露 `<audio-eq>` 标签 + 底层 `AudioEQCore` JS 模块。

### 算法迁移时一并修复的 v1.0.0 已知 bug

1. LPF/HPF 在 `f >= sampleRate/2` 时 `tan(π·f/fs)` 爆炸 → **加 Nyquist clamp**
2. Butterworth LPF/HPF 的 Q 硬编码为 `sqrt(2)` → **改为可配参数**
3. CalculateCore.cpp L142 用 `qmax=128.8`、L233 用 `qmax=128.0` 不一致 → **统一为 128.0**
4. CalculateCore.cpp L705-710 两分支代码完全相同 → **去掉死代码**

## Stage 拆分总览

> 各 Stage 的目标、SubStage 清单、交付物、验收标准与依赖关系，详见 [开发计划.md](./开发计划.md)。
>
> 每个 Stage 开始前，MainAgent 在 `StageX/` 目录下创建该 Stage 的 `设计文档.md` 与 `问题清单.md`，按规范第 6.1 节"零上下文可开发"标准要求详细化。

按代码依赖关系串行排列，每个 Stage 内部子任务完全并行。

```
Stage 1: 基础骨架与工程化
  ├── 1A: 项目骨架（package.json / Vite 配置 / ESM+UMD 双产物 / 目录结构）
  ├── 1B: 基础类型与常量（FilterType / SampleRate / EQBand / ShelfBand / 错误 code）
  └── 1C: CoordinateMapper（对数频率 + 线性增益双向映射）

Stage 2: 核心算法与数据模型（纯模块，无渲染）
  ├── 2A: ButterworthIIR（5 个 makeBiquad + evaluate + Nyquist clamp + 4 项 bug 修复）
  ├── 2B: EqualizerModel（Band / LPF / HPF / 采样率 / 焦点；EventTarget 事件）
  └── 2C: CurveEngine（同步对数频点生成 + 多 band 累加 + LPF/HPF 叠加）

Stage 3: Canvas2D 渲染层
  ├── 3A: CanvasRenderer 主控（dpr / clear / rAF / dirty flag）
  ├── 3B: GridLayer（回字形边距 + L 形轴 + 网格 + 0dB 高亮）
  ├── 3C: LabelLayer（频率/增益文字 + 边距对齐）
  ├── 3D: CurveLayer（主曲线 + 焦点单 band 曲线 + 半透明填充）
  └── 3E: HandleLayer（Band 圆形 + LPF/HPF 椭圆 + 内部文字）

Stage 4: 交互层
  ├── 4A: 命中测试 + 拖拽（Pointer Events + rAF 节流 + 60Hz）
  └── 4B: 焦点切换 + Z-Order

Stage 5: Custom Element 封装与对外 API
  ├── 5A: AudioEQElement（Shadow DOM / attribute / lifecycle / ResizeObserver / SSR 守卫）
  ├── 5B: 公共 JS API（addBand / removeBand / setSampleRate / events 等）
  └── 5C: AudioEQCore 头部模块导出 + JSDoc + `.d.ts` 生成

Stage 6: 示例与文档
  ├── 6A: SimpleDemo（HTML 单页面，覆盖所有 API 调用）
  └── 6B: README + 使用文档 + 发布说明

Stage 7: 测试与性能验收
  ├── 7A: 算法单元测试（ButterworthIIR 对比理论频响值；CoordinateMapper 边界）
  ├── 7B: Custom Element 集成测试（DOM + Pointer 模拟）
  └── 7C: 性能基准测试（10 band × 60Hz 拖拽 < 16ms 硬指标）
```

总计：**7 个 Stage，21 个 SubStage**。
