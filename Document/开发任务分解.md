## 二、Agent 任务分派

### deepseek-dev（擅长复杂逻辑、Vulkan、DSP、多线程、架构设计）

| 任务编号 | 任务名称 | 分派理由 |
|----------|----------|----------|
| A3 | FilterAlgorithm 抽象基类 + Factory | 涉及工厂模式设计、虚函数接口抽象，需要架构设计能力 |
| A4 | ButterworthIIR | 核心 DSP 算法，涉及双二阶滤波器数学推导与频响计算，需要信号处理专业背景 |
| A5 | CoordinateMapper | 对数/线性坐标映射，涉及频率标度变换和浮点精度处理，属于算法密集型 |
| A7 | CurveEngine | 多线程后台处理、对数频点生成、总曲线/单曲线叠加、取消机制，架构复杂度高 |
| B1 | VulkanContext | Vulkan 实例/逻辑设备/队列族创建，需要深入理解 Vulkan 初始化流程和错误处理 |
| B2 | VulkanSwapchain | MSAA 多重采样 + HDPI 高分辨率适配，涉及表面格式协商和重创建机制 |
| B3 | VulkanPipeline | 四条渲染管线 (Grid/Curve/Fill/Glyph) 的 Pipeline Layout、ShaderModule、RenderPass 配置 |
| B5 | GLSL Shader 编写 | 需要 GPU 着色器编程经验，涉及顶点变换、几何着色、片段处理 |
| B7 | VulkanFontAtlas | stb_truetype 字体光栅化 + 纹理图集打包，涉及 GPU 纹理上传和字距信息管理 |
| B8 | VulkanRenderer | 整合全部 Vulkan 子模块，设计渲染循环、Command Buffer 录制、资源生命周期管理 |
| B9 | 集成到 Qt Widget | QVulkanInstance + QWidget::createWindowContainer 桥接，需要 Qt/Vulkan 互操作经验 |
| C4 | Band 焦点态交互 | 涉及单 band 曲线着色、填充渐变、圆形高光渲染，需要 GPU 渲染和交互逻辑结合 |

**验收标准**
- [ ] `cmake -S . -B build` 在三大平台下均能通过 Configure 阶段

1. 使用 `cmake_minimum_required(VERSION 3.21)`，`project(AudioEQ LANGUAGES CXX)`
2. 各 target 使用 `add_library(... STATIC)` 或 `OBJECT` 库
3. 依赖查找使用 `find_package`，找不到时给出清晰的错误提示
4. 配置 Debug/Release 编译标志分离
5. 测试框架集成 (Google Test / Catch2)，通过 `FetchContent` 或 `find_package`

#### A3: 实现 FilterAlgorithm 抽象基类 + FilterAlgorithmFactory

| 属性 | 内容 |
|------|------|
| **编号** | A3 |
| **名称** | 实现 FilterAlgorithm 抽象基类 + FilterAlgorithmFactory |
| **负责人** | deepseek-dev |
| **预计工时** | 6h |
| **前置依赖** | A2 (Types.h) |
| **里程碑** | Phase 1-A 基础架构 |

**输入**
- A2 产出的 `Types.h`（FilterType、SampleRate 等）
- Spec 文档中 FilterAlgorithm 接口定义

**输出**
- `include/AudioEQ/FilterAlgorithm.h` — 抽象基类声明
- `src/audio/FilterAlgorithm.cpp` — 公共实现（含 Factory）
- `src/audio/FilterAlgorithmFactory.cpp` — 独立工厂实现

**验收标准**
- [ ] 抽象基类定义纯虚接口：
  - `virtual float evaluateAt(float frequency) const = 0`
  - `virtual std::pair<float, float> qRange() const = 0`
  - `virtual void setParams(const EQBand& band) = 0`
  - `virtual void reset() = 0`
  - `virtual std::unique_ptr<FilterAlgorithm> clone() const = 0`
- [ ] FilterAlgorithmFactory 根据 `FilterType` 和 `FilterAlgorithm` 创建具体实例
- [ ] Factory 支持注册自定义实现（可扩展）
- [ ] 析构函数为虚函数，确保安全多态删除
- [ ] 编译通过，可创建占位派生类进行接口验证

**实现要点**
1. 使用 CRTP 或纯虚接口模式，不混合数据成员
2. Factory 内部使用 `std::unordered_map` 映射 `(FilterType, FilterAlgorithm) → Creator`
3. 提供 `registerAlgorithm()` 静态方法用于运行时注册
4. `clone()` 返回 `unique_ptr` 支持 prototype 模式
5. `qRange()` 返回 `[最小有效Q, 最大有效Q]`，由子类实现具体范围

#### A4: 实现 ButterworthIIR

| 属性 | 内容 |
|------|------|
| **编号** | A4 |
| **名称** | 实现 ButterworthIIR |
| **负责人** | deepseek-dev |
| **预计工时** | 8h |
| **前置依赖** | A3 (FilterAlgorithm 基类) |
| **里程碑** | Phase 1-A 基础架构 |

**输入**
- A3 产出的 `FilterAlgorithm.h`（基类接口）
- A2 产出的 `Types.h`
- Butterworth 滤波器设计参考公式 (Analog Devices AN-649 / RBJ Audio EQ Cookbook)

**输出**
- `include/AudioEQ/ButterworthIIR.h`
- `src/audio/ButterworthIIR.cpp`

**验收标准**
- [ ] 继承 `FilterAlgorithm`，实现所有纯虚方法
- [ ] `evaluateAt(freq)` 返回正确的频率响应幅度 (dB)，误差 < 0.1 dB
- [ ] 支持 Peak/LPF/HPF/LSF/HSF 五种滤波器类型的 Butterworth 设计
- [ ] 内部 biquad 系数计算正确：
  - 使用双线性变换从 s 域映射到 z 域
  - 预畸变 (prewarping) 处理正确
- [ ] `setParams()` 输入参数合法性检查（频率 > 0, Q > 0 等）
- [ ] `reset()` 清除内部状态（延迟线清零）
- [ ] 线程安全：`evaluateAt` 可被多线程只读调用

**实现要点**
1. 使用直接形式 II (Direct Form II) 双二阶结构减少延迟单元
2. 系数计算函数独立为静态方法，方便单元测试
3. 使用 `double` 精度中间计算，输出转为 `float`
4. 关键公式注释标注参考文献出处
5. `qRange()` 返回 `[0.1, 40.0]` 作为 Butterworth 设计的 Q 值范围

- [ ] **析构竞态测试**：计算进行中销毁 CurveEngine，验证不 crash
- [ ] **焦点态切换测试**：在 `requestUpdate()` 和 `requestSingleCurve()` 之间快速切换
- [ ] **回调时序测试**：验证回调在主线程 / 正确的线程上下文中执行
- [ ] 使用 Thread Sanitizer (TSan) 验证无 data race
- [ ] 所有测试通过

**实现要点**
1. 使用 `std::barrier` (C++20) 或 `std::latch` 同步测试线程
2. 使用 `std::atomic<int>` 计数回调次数，验证取消后旧请求不回调
3. TSan 集成：CMake 中添加 `-fsanitize=thread` 编译选项
4. 析构竞态测试 pattern：
   ```cpp
   auto engine = std::make_unique<CurveEngine>(...);
   engine->requestUpdate();
   // 不等待完成，立即析构
   engine.reset();
   ```
5. 回调线程验证：使用 `QMetaObject::invokeMethod` 时检查 `QThread::currentThread()`
