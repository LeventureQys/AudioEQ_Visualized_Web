# AudioEQ 模块设计文档

## 模块总览

| 模块 | 职责 | 层级 |
|------|------|------|
| **AudioEQ** | 对外唯一入口，QWidget 子类，组合调度所有子模块 | 公共 API 层 |
| **ViewEqualizer** | 内部组合容器，协调 Vulkan 渲染区与交互覆盖层的布局与事件分发 | 交互层 |
| **EqualizerModel** | 数据模型，存储所有 Band/LPF/HPF 参数、采样率、焦点状态，变更时发出信号 | 数据层 |
| **CurveEngine** | 后台线程曲线计算引擎，异步生成总/单 Band 频响曲线点集 | 计算层 |
| **FilterAlgorithm** | 滤波器算法抽象基类，定义 evaluateAt() 纯虚函数 | 计算层 |
| **ButterworthIIR** | 二阶 IIR 巴特沃斯滤波器具体实现 | 计算层 |
| **FilterAlgorithmFactory** | 滤波器算法工厂，根据枚举创建对应算法实例 | 计算层 |
| **CoordinateMapper** | 频率/增益 ↔ 视口像素坐标的双向映射（对数+线性） | 计算层 |
| **BandHandle** | 普通 Band 交互控件，透明 QWidget，捕获鼠标拖拽和点击 | 交互层 |
| **LpfHandle** | LPF 椭圆手柄，锁定 0dB 增益，仅水平拖动 | 交互层 |
| **HpfHandle** | HPF 椭圆手柄，锁定 0dB 增益，仅水平拖动 | 交互层 |
| **VulkanRenderer** | Vulkan 渲染主控，整合所有 Vulkan 子模块，暴露绘制指令接口 | 渲染层 |
| **VulkanContext** | Vulkan 实例/物理设备/逻辑设备/队列的创建与销毁 | 渲染层 |
| **VulkanSwapchain** | 交换链 + MSAA 离屏渲染目标，高 DPI 适配 | 渲染层 |
| **VulkanPipeline** | 管理 4 条渲染管线（Grid/Curve/Fill/Glyph） | 渲染层 |
| **VulkanBufferPool** | VBO/IBO/UBO 的分配、更新与生命周期管理 | 渲染层 |
| **VulkanFontAtlas** | stb_truetype 光栅化字体纹理图集，管理字符字形数据 | 渲染层 |
| **VulkanFrameSync** | 帧同步原语（Fences/Semaphores），控制 CPU-GPU 并行度 | 渲染层 |

---

## 模块详细设计

### 1. AudioEQ

**职责**：对外唯一入口，QWidget 子类。组合持有 EqualizerModel、CurveEngine、ViewEqualizer，对外暴露全部配置 API，对内调度数据流和信号转发。

**接口/公开方法**：

```cpp
class AudioEQ : public QWidget {
    Q_OBJECT
public:
    static bool isVulkanSupported();

    explicit AudioEQ(QWidget* parent = nullptr);
    ~AudioEQ();

    int  bandCount() const;
    ResultCode setBandCount(int count);
    ResultCode addBand(const EQBand& band, int* outIndex = nullptr);
    ResultCode removeBand(int index);

    EQBand       bandParams(int index) const;
    ResultCode   setBandParams(int index, const EQBand& params);
    ResultCode   setBandFrequency(int index, double freqHz);
    ResultCode   setBandGain(int index, double gainDb);
    ResultCode   setBandQ(int index, double q);
    ResultCode   setBandType(int index, FilterType type);
    ResultCode   setBandAlgorithm(int index, FilterAlgorithm algo);
    ResultCode   setBandBypass(int index, bool bypass);

    int  focusedBandIndex() const;
    void setFocusedBandIndex(int index);

    SampleRate  sampleRate() const;
    ResultCode  setSampleRate(SampleRate rate);

    ResultCode setLpfEnabled(bool enabled);
    bool       isLpfEnabled() const;
    ResultCode setLpfFrequency(double freqHz);
    ResultCode setLpfAlgorithm(FilterAlgorithm algo);

    ResultCode setHpfEnabled(bool enabled);
    bool       isHpfEnabled() const;
    ResultCode setHpfFrequency(double freqHz);
    ResultCode setHpfAlgorithm(FilterAlgorithm algo);

    ResultCode setQRange(FilterType type, double min, double max);
    QPair<double,double> qRange(FilterType type) const;

    ResultCode setGainRange(double minDb, double maxDb);
    double gainMin() const;
    double gainMax() const;

    ResultCode setCurvePointCount(int count);
    int  curvePointCount() const;
    ResultCode setCurveColor(const QColor& color);
    ResultCode setBackgroundColor(const QColor& color);
    ResultCode setFrequencyVisibleRange(double minHz, double maxHz);

signals:
    void bandChanged(int index);
    void bandAdded(int index);
    void bandRemoved(int index);
    void bandSelected(int index);
    void bandDeselected();
    void focusedBandChanged(int index);
    void sampleRateChanged(SampleRate rate);
    void lpfChanged();
    void hpfChanged();
};
```

**内部数据结构**：

```cpp
private:
    EqualizerModel*    m_model;
    CurveEngine*       m_curveEngine;
    ViewEqualizer*     m_view;
    FilterAlgorithm*   m_filterAlgo;       // 共享给 CurveEngine 的滤波器实例（非拥有）
    QColor             m_curveColor;
    QColor             m_bgColor;
    double             m_gainMin = -48.0;
    double             m_gainMax = 48.0;
    QRect              m_viewport;
```

**依赖**：EqualizerModel、CurveEngine、ViewEqualizer、FilterAlgorithm、FilterAlgorithmFactory

**线程安全性**：所有公开 API 必须由主线程调用。内部通过 Qt 信号槽的 queued connection 与 CurveEngine 后台线程通信。

**设计要点**：
- 构造函数体内调用 `setBandCount(5)` 初始化默认 5 个 Band，默认频率按对数均匀分布在 [20Hz, 20kHz]
- 每项公开 API 调用在操作 model 后转发信号，并触发 CurveEngine 异步重算
- `isVulkanSupported()` 为静态方法，在构造 AudioEQ 前调用，内部尝试创建 QVulkanInstance 并查询物理设备
- 不直接管理 BandHandle / LpfHandle / HpfHandle / VulkanRenderer，这些由 ViewEqualizer 内部管理
- `setQRange()` 触发 model 自动 clamp 所有匹配类型的已有 Band 的 Q 值

### 2. EqualizerModel

**职责**：存储 EQ 全部配置参数（Band 列表、LPF/HPF、采样率、焦点状态、Q 值范围），变更时通过 Qt 信号通知订阅者，是数据流的唯一真相源。

**接口/公开方法**：

```cpp
class EqualizerModel : public QObject {
    Q_OBJECT
public:
    explicit EqualizerModel(QObject* parent = nullptr);

    int  bandCount() const;
    ResultCode setBandCount(int count);
    ResultCode addBand(const EQBand& band, int* outIndex = nullptr);
    ResultCode removeBand(int index);

    const EQBand& bandAt(int index) const;
    ResultCode setBandParams(int index, const EQBand& params);
    QVector<EQBand> allBands() const;

    int  focusedBandIndex() const;
    void setFocusedBandIndex(int index);
    ResultCode moveBandZOrder(int fromIndex, int toIndex);

    SampleRate sampleRate() const;
    ResultCode setSampleRate(SampleRate rate);
    double nyquistFrequency() const;

    ShelfBand lpf() const;
    ResultCode setLpf(const ShelfBand& lpf);
    ShelfBand hpf() const;
    ResultCode setHpf(const ShelfBand& hpf);
    ResultCode setLpfEnabled(bool enabled);
    ResultCode setHpfEnabled(bool enabled);

    ResultCode setQRange(FilterType type, double min, double max);
    QPair<double,double> qRange(FilterType type) const;

signals:
    void bandChanged(int index);
    void bandAdded(int index);
    void bandRemoved(int index);
    void bandCountChanged(int newCount);
    void focusedBandChanged(int index);
    void sampleRateChanged(SampleRate rate);
    void lpfChanged();
    void hpfChanged();
    void modelReset();
};
```

**内部数据结构**：

```cpp
private:
    QVector<EQBand>  m_bands;
    int              m_focusedBandIndex = -1;
    SampleRate       m_sampleRate = SampleRate::SR_44100;
    ShelfBand        m_lpf;
    ShelfBand        m_hpf;
    QMap<FilterType, QPair<double,double>> m_qRanges;  // 每种 FilterType 的 Q 值范围
```

**依赖**：无（仅依赖基础结构体定义：EQBand、ShelfBand、FilterType、SampleRate、ResultCode）

**线程安全性**：仅由主线程访问，自身不是线程安全的。CurveEngine 通过快照复制方式读取数据，不直接引用 model 内部状态。

**设计要点**：
- `addBand()` 忽略传入 EQBand 中的 index 字段，自动分配最小可用 index（0-based 连续分配），通过 `outIndex` 返回实际分配的 index。若已有 index 冲突返回 `IndexConflict`
- `removeBand()` 删除后列表中的 index 不重整（保持稀疏），后续 `addBand()` 会复用已释放的最小 index
- LPF index 固定为 -1，HPF index 固定为 -2，不混入普通 Band 列表
- `setSampleRate()` 变更后，X 轴最大刻度自动更新为 Nyquist，触发 CurveEngine 重算
- `setQRange()` 修改某类型的 Q 范围后，遍历所有 Band，对匹配类型的 Band 执行 `std::clamp` 其 Q 值，并通过 `bandChanged` 逐个通知
- `moveBandZOrder()` 交换 `m_bands` 中两个位置的元素，影响渲染时的 Band 绘制顺序

### 3. CurveEngine

**职责**：在独立后台线程中异步计算频响曲线点集。接收主线程的参数快照，通过 `FilterAlgorithm::evaluateAt()` 纯函数计算每个频率点的增益，结果通过信号发回主线程。

**接口/公开方法**：

```cpp
class CurveEngine : public QObject {
    Q_OBJECT
public:
    explicit CurveEngine(FilterAlgorithm* algo, QObject* parent = nullptr);
    ~CurveEngine();

    void setPointCount(int count);
    int  pointCount() const;
    void setFreqRange(double minHz, double maxHz);
    void setGainRange(double minDb, double maxDb);

public slots:
    void requestTotalCurve(const EqualizerModel& modelSnapshot);
    void requestSingleBandCurve(int bandIndex, const EqualizerModel& modelSnapshot);
    void cancelPending();

signals:
    void totalCurveReady(QVector<QPointF> points);
    void singleBandCurveReady(int bandIndex, QVector<QPointF> points);
};
```

**内部数据结构**：

```cpp
private:
    QThread              m_workerThread;
    FilterAlgorithm*     m_filterAlgo;            // 非拥有指针，纯函数，多线程共享无锁
    std::atomic<bool>    m_cancelPending{false};  // 原子标志，用于取消旧计算
    int                  m_pointCount = 500;
    double               m_freqMin = 1.0;
    double               m_freqMax = 24000.0;
    double               m_gainMin = -48.0;
    double               m_gainMax = 48.0;
```

**依赖**：FilterAlgorithm、EqualizerModel（值传递快照）、CoordinateMapper

**线程安全性**：自身运行在 `m_workerThread` 上。通过 `QObject::moveToThread()` 将工作对象移至 QThread。主线程通过 queued connection（Qt::QueuedConnection）发起计算请求。`FilterAlgorithm::evaluateAt()` 为 const 纯函数，所有线程共享同一实例无需加锁。`m_cancelPending` 为原子变量，无锁取消。

**设计要点**：
- 构造函数中将 `this` moveToThread(&m_workerThread)，槽函数在 worker 线程执行
- `requestTotalCurve()` 和 `requestSingleBandCurve()` 的参数 `modelSnapshot` 按值传递（`const EqualizerModel&` 由 Qt 元对象系统在跨线程排队时深拷贝），避免数据竞争
- 频点生成使用对数分布：`freq[i] = freqMin * (freqMax / freqMin)^(i / (pointCount - 1))`，保证每个倍频程采样密度一致
- 计算流程：生成对数频点 → 逐个频点累加所有非 bypass Band 的增益 → 叠加 LPF/HPF（若 enabled）→ CoordinateMapper 坐标变换 → 裁剪到增益范围 → 发射结果信号
- 取消机制：新的计算请求到达时，设置 `m_cancelPending = true`。计算循环中每 N 次迭代检查标志，若为 true 则提前退出
- resize 时不重算曲线，仅由 VulkanRenderer 通过 CoordinateMapper 重新映射视口坐标

### 4. FilterAlgorithm

**职责**：滤波器算法的抽象基类（非 QObject），定义滤波器的统一求值接口。所有实现必须保证 `evaluateAt()` 为纯函数。

**接口/公开方法**：

```cpp
class FilterAlgorithm {
public:
    virtual ~FilterAlgorithm() = default;

    virtual double evaluateAt(double freqHz, double sampleRate, const EQBand& band) const = 0;
    virtual FilterAlgorithmType type() const = 0;
    virtual QPair<double,double> qRange(FilterType filterType) const = 0;
};
```

**内部数据结构**：无状态成员，纯接口。

**依赖**：无（仅依赖 EQBand、FilterType 结构体定义）。

**线程安全性**：完全线程安全。`evaluateAt()` 为 const 纯函数，不依赖任何可变状态。多个线程可同时调用同一实例的 `evaluateAt()`，无需同步。

**设计要点**：
- 不暴露 biquad 系数（TFZ_coefficients），外部只通过 `(频率, 增益, Q值, 类型)` 配置
- `evaluateAt()` 的 `band` 参数包含该 Band 的完整配置（频率、增益、Q、类型），实现内部自行完成 biquad 计算和频率响应求值
- `qRange()` 返回指定 FilterType 下该算法支持的 Q 值范围
- 非 QObject，无需 Qt 元对象系统

### 5. ButterworthIIR

**职责**：FilterAlgorithm 的二阶 IIR 巴特沃斯具体实现，支持 Peak/LowShelf/HighShelf/LowPass/HighPass/BandPass 六种滤波器类型。

**接口/公开方法**：

```cpp
class ButterworthIIR : public FilterAlgorithm {
public:
    FilterAlgorithmType type() const override { return FilterAlgorithmType::ButterworthIIR; }
    double evaluateAt(double freqHz, double sampleRate, const EQBand& band) const override;
    QPair<double,double> qRange(FilterType filterType) const override;
};
```

**内部数据结构**：

```cpp
private:
    struct BiquadCoeff { double b0, b1, b2, a1, a2; };

    BiquadCoeff makePeakFilter(double freq, double q, double gain, double sampleRate) const;
    BiquadCoeff makeLowShelf(double freq, double q, double gain, double sampleRate) const;
    BiquadCoeff makeHighShelf(double freq, double q, double gain, double sampleRate) const;
    BiquadCoeff makeLowPass(double freq, double sampleRate) const;
    BiquadCoeff makeHighPass(double freq, double sampleRate) const;

    double freqResponseGain(const BiquadCoeff& coeff, double freq, double sampleRate) const;
```

**依赖**：FilterAlgorithm（继承）

**线程安全性**：完全线程安全。所有方法为 const，无状态成员变量。内部 biquad 系数在栈上计算，不存储。

**设计要点**：
- `evaluateAt()` 实现：根据 `band.type` 选择对应的 `make*` 方法计算 Biquad 系数 → 调用 `freqResponseGain()` 求指定频率处的增益(dB)
- `freqResponseGain()` 的数学公式：`10 * log10(numerator / denominator)`，其中 numerator/denominator 由 biquad 系数和频率计算
- Q 范围：Peak/LowPass/HighPass/BandPass → [0.4, 128.0]；LowShelf/HighShelf → [0.4, 1.6]
- makePeakFilter / makeLowShelf / makeHighShelf 的数学公式直接迁移自 `Document/basic_code/CalculateCore.cpp` 中的对应函数，仅将 `PublicVar::ins().sample_rate` 替换为参数 `sampleRate`

### 6. FilterAlgorithmFactory

**职责**：根据 `FilterAlgorithmType` 枚举创建对应的算法实例，支持运行时注册自定义算法。

**接口/公开方法**：

```cpp
class FilterAlgorithmFactory {
public:
    static std::unique_ptr<FilterAlgorithm> create(FilterAlgorithmType algoType);
    static void registerAlgorithm(FilterAlgorithmType algoType,
                                  std::function<std::unique_ptr<FilterAlgorithm>()> creator);
};
```

**内部数据结构**：

```cpp
private:
    static std::map<FilterAlgorithmType,
                    std::function<std::unique_ptr<FilterAlgorithm>()>> s_creators;
```

**依赖**：FilterAlgorithm

**线程安全性**：`registerAlgorithm()` 非线程安全，应在程序初始化阶段单线程调用。`create()` 为只读查找，初始化完成后线程安全。

**设计要点**：
- 内置注册表在静态初始化阶段写入 `{ButterworthIIR → [](){ return std::make_unique<ButterworthIIR>(); }}`
- `create()` 查找失败时返回 `nullptr`
- `registerAlgorithm()` 允许外部在构造 AudioEQ 前注册自定义算法实现

### 7. CoordinateMapper

**职责**：频率/增益 ↔ 视口像素坐标的双向映射。频率轴使用对数映射，增益轴使用线性映射。

**接口/公开方法**：

```cpp
class CoordinateMapper {
public:
    CoordinateMapper(QRect viewport, double freqMin, double freqMax, double gainMin, double gainMax);

    double freqToX(double freqHz) const;
    double gainToY(double gainDb) const;
    double xToFreq(double x)      const;
    double yToGain(double y)      const;

    void setViewport(QRect viewport);
    void setFreqRange(double min, double max);
    void setGainRange(double min, double max);

    double freqMin() const;
    double freqMax() const;
    double gainMin() const;
    double gainMax() const;
    QRect  viewport() const;
};
```

**内部数据结构**：

```cpp
private:
    QRect  m_viewport;
    double m_freqMin, m_freqMax;   // Hz
    double m_gainMin, m_gainMax;   // dB
    double m_logFreqRatio;         // log(freqMax / freqMin)，预计算避免重复 log
```

**依赖**：无

**线程安全性**：线程安全。所有方法为 const，setter 方法在 UI 线程调用。

**设计要点**：
- 频率对数映射公式：`x = viewport.x + viewport.w * log(freq / freqMin) / log(freqMax / freqMin)`，clamp 到视口范围内
- 增益线性映射公式：`y = viewport.y + viewport.h * (gainMax - gainDb) / (gainMax - gainMin)`
- 反向映射 `xToFreq()`：`freq = freqMin * pow(freqMax / freqMin, (x - viewport.x) / viewport.w)`
- `setViewport()` 在 resize 时调用，仅更新视口矩形，不触发曲线重算
- 构造时预计算 `m_logFreqRatio = log(freqMax / freqMin)` 加速后续转换

### 8. BandHandle

**职责**：透明 QWidget 覆盖层，表示一个普通 EQ Band。捕获鼠标点击（焦点切换）和拖拽事件（调整频率/增益），将交互结果通过信号发出。

**接口/公开方法**：

```cpp
class BandHandle : public QWidget {
    Q_OBJECT
public:
    explicit BandHandle(int bandIndex, const CoordinateMapper& mapper, QWidget* parent = nullptr);

    void updatePosition(double freqHz, double gainDb);
    bool isFocused() const;
    void setFocused(bool focused);

signals:
    void bandDragged(int index, double newFreq, double newGain);
    void bandPressed(int index);
    void bandDeselected();
};
```

**内部数据结构**：

```cpp
private:
    int                m_bandIndex;
    const CoordinateMapper& m_mapper;
    QPointF            m_centerPos;       // 视口像素坐标
    bool               m_focused = false;
    bool               m_dragging = false;
    QTimer             m_dragTimer;       // 60Hz 节流定时器，16ms 间隔
    QPointF            m_pendingPosition; // 拖拽中的待处理位置
    static constexpr int HANDLE_RADIUS = 8;  // 圆形手柄半径(px)，需 × DPI 缩放
```

**依赖**：CoordinateMapper

**线程安全性**：仅由主线程（UI 线程）操作，Qt 信号在主线程发射。

**设计要点**：
- 透明覆盖层，大小与父控件 ViewEqualizer 相同（通过布局撑满），通过 `setAttribute(Qt::WA_TransparentForMouseEvents, false)` 使控件接受鼠标事件，但 `paintEvent` 留空（圆形由 VulkanRenderer 绘制）
- 鼠标点击判断是否命中：计算鼠标位置与 `m_centerPos` 的距离，小于 `HANDLE_RADIUS * devicePixelRatio` 视为命中
- 拖拽节流：`mouseMoveEvent` 中更新 `m_pendingPosition`，启动 16ms QTimer。定时器触发时执行实际坐标转换并发射 `bandDragged` 信号
- 拖拽帧率上限 60Hz（16.67ms），避免无效高频更新
- `updatePosition()` 由外部（AudioEQ 槽函数）调用，将频率/增益转换为像素坐标更新 `m_centerPos`，并调用 `move()` 微调 Widget 位置（或仅记录坐标供命中测试用）
- 焦点切换：`setFocused(true)` 时改变视觉状态（由 VulkanRenderer 根据焦点状态渲染不同颜色），`setFocused(false)` 时恢复默认

- 帧提交：`requestRedraw()` 设置 `m_pendingRedraw = true`，下一帧 `renderFrame()` 检查标志并重建 CommandBuffer
- 空闲态使用 `VK_PRESENT_MODE_FIFO`（VSync），拖拽态通过 QTimer 限制提交频率 ≤ 60Hz

### 14. VulkanPipeline

**职责**：管理 4 条渲染管线的创建、切换和销毁：Grid（网格线）、Curve（曲线）、Fill（填充）、Glyph（文字/手柄）。每条管线包含独立的 Shader Module、Pipeline Layout、Descriptor Set Layout。

**接口/公开方法**：

```cpp

    Curve,
    Fill,
    Glyph

    VkDescriptorSetLayout descriptorSetLayout(VulkanPipelineType type) const;

    struct PipelineData {
        VkPipeline            pipeline = VK_NULL_HANDLE;
        VkPipelineLayout      layout = VK_NULL_HANDLE;
        VkDescriptorSetLayout descSetLayout = VK_NULL_HANDLE;
```

**依赖**：VulkanContext

**线程安全性**：初始化阶段主线程调用，渲染阶段在命令录制线程访问（只读）。

**设计要点**：

| 管线 | 图元拓扑 | Vertex Shader 输入 | Fragment Shader 特效 |
|------|---------|-------------------|---------------------|
| **Grid** | `VK_PRIMITIVE_TOPOLOGY_LINE_STRIP` | 2D 像素坐标 | 固定颜色 + alpha |
| **Curve** | `VK_PRIMITIVE_TOPOLOGY_TRIANGLE_STRIP` | 2D 像素坐标 + 曲线参数 | smoothstep 抗锯齿边缘 |
| **Fill** | `VK_PRIMITIVE_TOPOLOGY_TRIANGLE_STRIP` | 2D 像素坐标 | 半透明填充 `rgba(0,255,0,0.15)` |
| **Glyph** | `VK_PRIMITIVE_TOPOLOGY_TRIANGLE_STRIP` | 2D 像素坐标 + UV | 纹理采样 + alpha 混合 |

- Grid 管线：顶点数据为网格线段的端点（横线 ×N + 竖线 ×M），通过 VBO 上传
- Curve 管线：顶点数据为 Catmull-Rom 细分后的 TriangleStrip。Fragment Shader 中使用 `smoothstep` 做曲线边缘抗锯齿（主曲线 2px，单 Band 曲线 1.5px）
- Fill 管线：顶点数据为曲线下方的填充区域（曲线点 + 底部直线段构成的 TriangleStrip）
- Glyph 管线：每个字形为 2 个三角形（Quad），顶点包含 position + UV。绑定 VulkanFontAtlas 的纹理 DescriptorSet
- Descriptor Set Layout：Curve/Fill 管线使用 Uniform Buffer（MVP 矩阵），Glyph 管线使用 Combined Image Sampler（字体纹理）

### 15. VulkanBufferPool

**职责**：管理所有 Vulkan 缓冲区（VBO、IBO、UBO）的分配、映射更新和生命周期。支持频繁更新的动态缓冲区（曲线 VBO、文字 VBO）和静态缓冲区（网格线 VBO）。

**依赖**：VulkanContext

**线程安全性**：`allocate*()` 和 `updateBuffer()` 在初始化阶段和 UI 更新阶段由主线程调用。`beginFrame()` / `endFrame()` 在渲染线程调用。`updateBuffer()` 使用双缓冲或 staging buffer 避免 GPU 正在使用时被覆写。

**设计要点**：
- 动态缓冲区（曲线 VBO、文字 VBO）：使用 `VK_MEMORY_PROPERTY_HOST_VISIBLE_BIT | VK_MEMORY_PROPERTY_HOST_COHERENT_BIT`，支持频繁 `vkMapMemory` / `memcpy` / `vkUnmapMemory`
- 静态缓冲区（网格线 VBO/IBO）：使用 `VK_MEMORY_PROPERTY_DEVICE_LOCAL_BIT`，通过 staging buffer 一次性上传
- UBO：每个 swapchain image 一份，避免帧间数据冲突。大小约 64 bytes（MVP 矩阵 = 4×4 float）
- 曲线 VBO 大小估算：`(pointCount * catmullRomSubdivFactor) * sizeof(Vertex)`，其中 Vertex 包含 `{float x, y, u}` 约 12 bytes
- 支持从大块 VkDeviceMemory 中做子分配（offset + size），减少碎片

### 17. VulkanFrameSync

**职责**：管理帧同步原语（Fences、Semaphores），控制 CPU-GPU 并行度，防止 command buffer 覆盖、保证 swapchain image 正确释放。

**接口/公开方法**：

```cpp
class VulkanFrameSync {
public:
    ResultCode initialize(VulkanContext* ctx, uint32_t maxFramesInFlight);
    void shutdown();

    void waitForFrame();
    void submitFrame(VkCommandBuffer cmd, VkSemaphore waitSemaphore,
                     VkSemaphore signalSemaphore);

    VkSemaphore imageAvailableSemaphore() const;
    VkSemaphore renderFinishedSemaphore() const;
    VkFence     currentFence() const;

    void advanceFrame();

private:
    struct FrameSyncData {
        VkFence     fence;
        VkSemaphore imageAvailable;
        VkSemaphore renderFinished;
    };
    QVector<FrameSyncData> m_frames;
    uint32_t              m_maxFramesInFlight = 2;
    uint32_t              m_currentFrame = 0;
    VulkanContext*        m_context = nullptr;
};
```

**依赖**：VulkanContext

**线程安全性**：渲染线程调用 `waitForFrame()` / `submitFrame()` / `advanceFrame()`。

**设计要点**：
- `maxFramesInFlight = 2`（双缓冲），允许 GPU 处理当前帧时 CPU 准备下一帧
- 每帧流程：
  1. `waitForFrame()`：`vkWaitForFences(m_frames[m_currentFrame].fence)` 等待上一轮该槽位的 GPU 工作完成，然后 `vkResetFences`
  2. `vkAcquireNextImageKHR(..., m_frames[m_currentFrame].imageAvailable, ...)`
  3. 录制 command buffer
  4. `submitFrame()`：`vkQueueSubmit` 等待 `imageAvailable` semaphore，完成后 signal `renderFinished` semaphore + fence
  5. `vkQueuePresentKHR` 等待 `renderFinished` semaphore
  6. `advanceFrame()`：`m_currentFrame = (m_currentFrame + 1) % m_maxFramesInFlight`
- Semaphores 用于 GPU-GPU 同步（swapchain image 可用 → 渲染完成），Fences 用于 CPU-GPU 同步（等待帧槽位释放）

```
  VulkanRenderer 拥有 VulkanContext、VulkanSwapchain、VulkanPipeline、
                  VulkanBufferPool、VulkanFontAtlas、VulkanFrameSync
  CurveEngine 依赖 FilterAlgorithm、CoordinateMapper、EqualizerModel (快照)
  BandHandle / LpfHandle / HpfHandle 依赖 CoordinateMapper
```

## 关键设计约束

### 1. 无单例
- 全库禁止任何单例模式。每个 AudioEQ 实例独立创建和管理全部资源
- `FilterAlgorithmFactory` 使用静态注册表但无实例依赖（算法实例可安全共享，因为 `evaluateAt()` 为纯函数）
- 原始参考代码中的 `CalculateCore::ins()` 单例、`PublicVar::ins()` 单例均废弃

### 2. FilterAlgorithm::evaluateAt() 纯函数
- `evaluateAt()` 声明为 `const virtual`，不依赖任何实例内部状态
- 所有 biquad 系数在函数内部的栈上计算，不存储成员变量
- 多线程可同时调用同一 FilterAlgorithm 实例，天然线程安全，无需加锁

### 3. QObject 信号槽跨线程通信
- CurveEngine 通过 `QObject::moveToThread()` 移至后台线程
- 主线程 → CurveEngine 的请求使用 `Qt::QueuedConnection`（自动，因为跨线程）
- CurveEngine → 主线程的结果信号同样为 queued，参数由 Qt 元对象系统自动深拷贝
- `EqualizerModel` 按值传递快照给 CurveEngine，避免数据竞争

### 4. Vulkan 多实例独立
- 每个 AudioEQ 拥有独立的 `VulkanContext`（Instance + Device + Queue）
- 同一进程内多个 AudioEQ 实例的 Vulkan 资源完全隔离
- 每个实例创建独立的 `QVulkanInstance` 和 `QWindow`（通过 `QWidget::createWindowContainer` 嵌入）
- `AudioEQ::isVulkanSupported()` 为静态方法，在构造任何实例前调用

### 5. Catmull-Rom 曲线平滑
- CurveEngine 输出离散采样点（默认 500 点）
- GPU 端接收离散点后进行 Catmull-Rom 样条插值细分
- 细分密度自适应视口宽度：约每像素 1-2 个细分点
- 生成 TriangleStrip 送入 Vulkan，Fragment Shader 中使用 `smoothstep` 做距离衰减抗锯齿
- 主曲线线宽 2px，单 Band 曲线线宽 1.5px（均乘以 DPI 缩放）

### 6. 对数频点分布
- CurveEngine 计算总曲线时，频点按对数均匀分布生成
- 公式：`freq[i] = freqMin * (freqMax / freqMin)^(i / (pointCount - 1))`
- 效果：每个倍频程（octave）拥有相同数量的采样点，低频段不会稀疏、高频段不会过密
- X 轴显示范围从 10Hz 到 Nyquist 频率
- CoordinateMapper 频率 → X 坐标映射使用 `log(freq / freqMin) / log(freqMax / freqMin)` 公式

### 7. 帧率控制
- **空闲态**：交换链使用 `VK_PRESENT_MODE_FIFO`（强制 VSync），跟随显示器刷新率
- **拖拽态**：BandHandle/LpfHandle/HpfHandle 使用 QTimer 16ms 间隔节流，限制拖拽更新频率 ≤ 60Hz
- 拖拽时：第一个事件立即处理（降低延迟），后续事件由定时器批量处理
- CurveEngine 取消机制：新计算请求到达时取消旧的未完成计算（通过 `std::atomic<bool>` 原子标志），确保拖拽时始终处理最新状态

### 8. Widget resize 优化
- `ViewEqualizer::resizeEvent` 中仅更新 `CoordinateMapper::setViewport()` 和 `VulkanSwapchain::resize()`
- 不触发 CurveEngine 重算（曲线频点不变，仅视口映射变化）
- VulkanRenderer 在 resize 后使用更新后的 MVP 矩阵重新渲染

### 9. 颜色硬编码 + 可配置覆盖
- 默认颜色方案硬编码（绿色曲线 `#00FF00`、暗灰背景 `#1A1A1A` 等）
- AudioEQ 提供 `setCurveColor()` 和 `setBackgroundColor()` 允许外部覆盖
- Band 圆形/椭圆、文字、网格线的颜色保持硬编码（确保 UI 一致性）

### 10. API 返回值与错误处理
- 所有状态变更 API 返回 `ResultCode` 枚举
- 不抛异常，调用方检查返回值处理错误
- 非法参数返回 `InvalidParameter`，索引越界返回 `IndexOutOfRange`
- Vulkan 初始化失败在 `initialize()` 返回 `VulkanNotAvailable`，调用方须在 `AudioEQ` 构造函数中处理
