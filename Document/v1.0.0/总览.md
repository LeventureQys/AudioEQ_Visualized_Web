## 全局技术决策（所有 Stage 必须遵守）

> 以下为 [问题清单.md](./问题清单.md) Q1–Q18 关闭后形成的最终技术基线。任何 Stage/SubStage 实现必须与此一致，不得自行偏离。

| 领域 | 决策 | 来源 |
|------|------|------|
| 目标平台 | Windows + Linux 完整支持；**macOS 仅保证编译通过 + portability flag，真机验证推迟** | Q1/Q14 |
| 库类型 | CMake 选项控制，默认动态库（`BUILD_SHARED_LIBS=ON`），需要 `AUDIOEQ_EXPORT` 导出宏 | Q3 |
| 第三方依赖 | 直接内置于 `thirdparty/`（见下方清单） | Q2 |
| **Vulkan 加载** | **使用 volk + 内置 vulkan-headers，不依赖系统 Vulkan SDK**。所有 Vulkan 代码 `#include "volk.h"`（不直接 include `<vulkan/vulkan.h>`），启动时 `volkInitialize()` | Q11 |
| **Vulkan instance** | **Qt 创建 VkInstance，再用 `volkLoadInstance(qtInstance->vkInstance())` 让 volk 加载函数指针** | Q12 |
| **Shader 工作流** | **GLSL 预编译为 `.spv` 内置进仓库**，CMake 把 `.spv` 转字节数组编入库；构建机无需任何 GLSL 编译器 | Q9/Q13 |
| Qt-Vulkan 集成 | 自定义 `QWindow`（`QSurface::VulkanSurface`）+ `QVulkanInstance::surfaceForWindow()` + `createWindowContainer`，**不使用 QVulkanWindow** | Q8 |
| 测试框架 | gtest。**先尝试 git submodule**（`thirdparty/code/googletest`）；submodule 行不通则退回直接内置源码 | Q4/Q15 |
| 字体 | 把 `thirdparty/font/Noto_Sans/static/NotoSans-Regular.ttf` 转 C 字节数组头文件编入库，**不用 Qt 资源系统/磁盘路径** | Q16 |
| 数学库 | **GLM 仅用于 Vulkan 渲染层**（MVP/NDC）；业务层与对外接口保持 Qt 类型（QPointF/QRectF），边界处转换一次 | Q17 |
| 滤波算法 | 提取 `Document/basic_code/CalculateCore.cpp` 的数学公式，重写为**无状态** ButterworthIIR，剥离单例/全局依赖 | Q18 |
| 颜色 | 曲线 `#00FF00`、背景 `#1A1A1A`，硬编码默认值，接口可覆盖 | Q7 |
| SampleRateManager / BandFocusManager | **不作为独立类**，功能内嵌 EqualizerModel | Q5/Q6 |
| SimpleDemo | 独立可执行文件，单独 CMake target | Q10 |

## 项目背景（新 Session 必读）

本项目是一个以**动态库**形式交付的音频 EQ 可视化 SDK：
- 对外暴露唯一入口类 `AudioEQ`（`QWidget` 子类）
- 用 **Vulkan** 渲染频响曲线、网格、文字（通过 **volk** 加载，不依赖系统 SDK）
- 用 **Qt Widget** 层处理用户的鼠标拖拽交互
- 内部使用后台线程（`CurveEngine`）异步计算频响曲线，不阻塞 GUI
- 构建系统：**CMake 3.20+**，语言：**C++17**，Qt：**Qt6 Widgets**

详细设计见：
- [模块设计文档.md](../模块设计文档.md) — 每个类的职责和接口
- [数据流设计文档.md](../数据流设计文档.md) — 信号/槽流向和线程模型
- [第一阶段开发文档-spec.md](../第一阶段开发文档-spec.md) — 数据结构定义和所有已确认决策

## Stage 规划总览

按代码依赖关系串行排列，每个 Stage 内部子任务完全并行。

```
Stage1（基础骨架）
  ├── 1A: CMake 项目骨架
  ├── 1B: 基础枚举和数据结构（AudioEQTypes.h）
  ├── 1C: CoordinateMapper
  └── 1D: GLSL Shader 源码（4 组）

Stage2（核心无依赖模块）
  ├── 2A: FilterAlgorithm 抽象基类 + FilterAlgorithmFactory
  ├── 2B: EqualizerModel（采样率/焦点逻辑内嵌，不建独立 Manager 类）
  ├── 2C: VulkanContext + isVulkanSupported()
  └── 2D: BandHandle（鼠标交互 Widget）

Stage3（依赖 Stage2 的模块）
  ├── 3A: ButterworthIIR（依赖 2A）
  ├── 3B: CurveEngine（依赖 2A, 2B, 1C）
  ├── 3C: VulkanSwapchain + VulkanFrameSync（依赖 2C）
  ├── 3D: VulkanPipeline + SPIR-V 编译集成（依赖 2C, 1D）
  ├── 3E: VulkanBufferPool（依赖 2C）
  ├── 3F: VulkanFontAtlas（依赖 2C）
  └── 3G: LpfHandle + HpfHandle（依赖 2D）

Stage4（Vulkan 全集成）
  └── 4A: VulkanRenderer（依赖 3C, 3D, 3E, 3F）

Stage5（Qt-Vulkan 桥接 + 顶层集成）
  ├── 5A: ViewEqualizer + Qt-Vulkan 集成（依赖 4A, 2B, 2D, 3G）
  └── 5B: AudioEQ 顶层类（依赖 5A, 3B）

Stage6（示例程序）
  └── 6A: SimpleDemo（依赖 5B）

Stage7（单元测试）
  ├── 7A: ButterworthIIR 单元测试（依赖 3A）
  ├── 7B: CoordinateMapper 单元测试（依赖 1C）
  └── 7C: CurveEngine 多线程安全测试（依赖 3B）

Stage8（兼容性测试）
  └── 8A: 兼容性/集成测试（依赖 5B）

Stage9（SimpleDemo 全接口交互面板）
  ├── 9A: 顶部控件（采样率 + 全局滤波器）+ 主框架重构（依赖 5B）

Stage10（回字形边距布局）

  ├── 11B: 网格步长与档位（gain 6 dB / freq 9 档 / 0 dB 独立 VBO）（依赖 10A）
  ├── 11C: 文字布局（频率贴 0 dB / 响度无 +dB / 数字基线对齐 / Hz/KHz 大写）（依赖 10A）
```
