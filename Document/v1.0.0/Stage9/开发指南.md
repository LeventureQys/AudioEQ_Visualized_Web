## 2. 跨 SubStage 接口契约（绝对源)

> 这一节是"零上下文可开发"的核心。**所有 SubAgent 在编码时必须严格遵守以下定义**;任何偏离都属于擅自修改任务边界,需通过 MainAgent 升级到问题清单。

### 2.1 `DemoMainWindow` 公共接口 (由 9A 提供给 9B / 9C 使用)

```cpp
// examples/SimpleDemo/DemoMainWindow.h
#pragma once
#include <QMainWindow>
#include "AudioEQ.h"

class QLabel;
class BandInspectorPanel;   // 由 9B 实现
class BandsTablePanel;      // 由 9C 实现

class DemoMainWindow : public QMainWindow {
    Q_OBJECT
public:
    explicit DemoMainWindow(QWidget* parent = nullptr);
    ~DemoMainWindow() override;

    // SDK 句柄,9B/9C 通过它访问所有 AudioEQ 状态
    AudioEQ* eq() const { return m_eq; }

    // StatusBar 协议: 按 key 维护多段文字,setStatusMessage 写 / 覆盖 / 清空 (value 为空串时清空)
    void setStatusMessage(const QString& key, const QString& value);

    // 统一 ResultCode 上报: rc != OK 时在 StatusBar lastResult 段显示红色 "<op>: <code>"
    // 用法: apply(eq->setBandFrequency(idx, hz), QStringLiteral("setBandFrequency"));
    void apply(ResultCode rc, const QString& op);

private:
    AudioEQ*               m_eq        = nullptr;
    BandInspectorPanel*    m_inspector = nullptr;  // 9B 注入
    BandsTablePanel*       m_table     = nullptr;  // 9C 注入
    QMap<QString, QLabel*> m_statusLabels;
};
```

**注入约定**：9B 与 9C 完成各自 Panel 后,在 `DemoMainWindow` 构造函数末尾通过 `addDockWidget` 加入并保存指针。9A 仅需为这两个成员**留位**和**显式 `nullptr` 初始化**,不需要为 Panel 写任何业务代码。

### 2.2 StatusBar Key 表 (源自设计文档第 8 节,锁定值)

| Key            | 文字格式 (示例)              | 写入方  | 触发时机                                  |
|----------------|----------------------------|--------|------------------------------------------|
| `focused`      | `focusedBand=#3`           | 9B     | 监听 `AudioEQ::focusedBandChanged`        |
| `count`        | `bandCount=5`              | 9C     | 见第 4 节缺口 (signals 待澄清)            |
| `sr`           | `sampleRate=48000`         | 9A     | 监听 `AudioEQ::sampleRateChanged`         |
| `gainRange`    | `gainRange=[-12,+12]`      | 9B     | Gain Range SpinBox 改值后                  |
| `qRange`       | `qRange(Peak)=[0.1,18]`    | 9B     | 焦点 band Type 切换 / Q Range 改值后        |
| `points`       | `pointCount=500`           | 9B     | Point Count SpinBox 改值后                |
| `lastResult`   | `OK` 或 `setBandQ: InvalidParameter` (红色) | 9A 提供 `apply()`,9A/9B/9C 都通过它写 | 任何 `ResultCode` 返回时 |

> StatusBar 由 9A 在构造时全部用 `QLabel` 初始化并 `addPermanentWidget` 加入 `statusBar()`,9B/9C 只通过 `setStatusMessage()` 改文字。

### 2.3 信号回环防护规范 (写入代码时必读)

设计文档第 9.2 项已点名"UI ↔ SDK 双向同步是经典坑点"。本指南把规范锁定为:

1. **永远以 SDK 状态为唯一真理来源** ([设计文档.md](./设计文档.md) "黄金法则")。控件值不缓存,信号到达后重读 SDK 即可。
2. **刷新 UI 控件时,统一使用 `QSignalBlocker`**,不要混用 `disconnect/connect` 或手工 `bool m_updating` flag,避免遗漏路径。示例:
   ```cpp
   void BandInspectorPanel::refreshFromSdk(int idx) {
       const QSignalBlocker b1(m_typeCombo);
       const QSignalBlocker b2(m_freqSpin);
       const QSignalBlocker b3(m_freqSlider);
       // ... 其余控件
       EQBand band = m_eq->bandParams(idx);
       m_typeCombo->setCurrentIndex(static_cast<int>(band.type));
       m_freqSpin->setValue(band.freqHz);
       // QSignalBlocker 在作用域结束时自动恢复
   }
   ```
3. **嵌入到 `QTableWidget` 单元格中的 `QComboBox` / `QCheckBox` 同样**——9C 在重建/局部刷新表格时,必须先 `QSignalBlocker` 包裹再写值。
4. **SDK setter 调用前不要清空控件状态**,调用后等 SDK 信号回流再刷新 UI——确保 UI 显示的永远是 SDK 已应用的值,而不是"我们打算让它变成什么"。

### 2.4 SubAgent 间禁止直接通信

按 [Agent开发规范](../../Agent开发规范.md) 第 2 节,9A/9B/9C 之间**不**通过函数调用、共享头文件以外的方式互访。所有需要协作的数据流都通过:
- `AudioEQ` 公共信号 (`bandParamsChanged` / `focusedBandChanged` / `sampleRateChanged`,以及第 4 节待澄清的 `bandAdded` / `bandRemoved`)
- `DemoMainWindow::setStatusMessage(key, value)`

如果发现需要新加跨 SubStage 的协作通道(例如 9B 想直接调 9C 的方法刷新表格),**停止**,通过 MainAgent 把问题追加到 [问题清单.md](./问题清单.md)。

     // 成员: QVector<bool> m_bypassAllSnapshot;
     void onBypassAllToggled(bool on) {
         int n = m_eq->bandCount();
         if (on) {
             m_bypassAllSnapshot.clear();
             m_bypassAllSnapshot.reserve(n);
             for (int i = 0; i < n; ++i) {
                 m_bypassAllSnapshot.append(m_eq->bandParams(i).bypass);
                 apply(m_eq->setBandBypass(i, true), "setBandBypass");
             }
         } else {
             // 解勾时按快照逐项还原;若 bandCount 在解勾期间变了,只还原 min(n, snapshot.size())
             int restore = std::min(n, m_bypassAllSnapshot.size());
             for (int i = 0; i < restore; ++i) {
                 apply(m_eq->setBandBypass(i, m_bypassAllSnapshot[i]), "setBandBypass");
             }
             m_bypassAllSnapshot.clear();
         }
     }

   - Remove Band 按钮:`apply(m_eq->removeBand(focused), "removeBand")`。

       EQBand b;
       b.type   = static_cast<FilterType>(m_addType->currentIndex());
       b.freqHz = m_addFreq->value();
       b.gainDb = m_addGain->value();
       b.q      = m_addQ->value();
       int newIdx = -1;
       ResultCode rc = m_eq->addBand(b, &newIdx);
       m_owner->apply(rc, QStringLiteral("addBand"));
       if (rc == ResultCode::OK && newIdx >= 0) {
           m_eq->setFocusedBandIndex(newIdx);
       }
   }
   ```

    void onTableSelectionChanged();
    void onCellComboTypeChanged(int row, int comboIdx);
    void onCellBypassToggled(int row, bool checked);
    void onContextMenuRequested(const QPoint& pos);

private:
    void rebuildTable();                       // 整表重建
    void refreshRow(int row);                  // 单行重读 SDK
    DemoMainWindow* m_owner;
    AudioEQ*        m_eq;
    QSpinBox*       m_bandCountSpin = nullptr;
    QTableWidget*   m_table         = nullptr;
};
```

**开发步骤**:

1. **Bands Count SpinBox** (设计文档把它放在 BottomPanel 顶部):范围 0–32 步长 1,`valueChanged(int)` → `apply(setBandCount(n), "setBandCount")`,改后写 `setStatusMessage("count", ...)`。
2. **Bands 表格**:`QTableWidget` 6 列 (`#`, `Type`, `Freq Hz`, `Gain dB`, `Q`, `Bypass`):
   - `#` 列:只读 `QTableWidgetItem`,显示 band 索引。
   - `Type` 列:`setCellWidget(row, 1, new QComboBox)`,填入 5 个 `FilterType`,变更 → `apply(setBandType(row, t), "setBandType")`。
   - `Freq Hz / Gain dB / Q` 列:只读,显示数值 (格式 `%.1f` / `%.2f` 视情况)。这三列**参数编辑统一走 RightPanel**,表格不开放直接编辑——这是设计文档的明确划分。
   - `Bypass` 列:`setCellWidget(row, 5, new QCheckBox)`,`toggled(bool)` → `apply(setBandBypass(row, b), "setBandBypass")`。
   - 单击行 → `m_eq->setFocusedBandIndex(row)`。
3. **行右键菜单**:`Remove Band` / `Duplicate Band`。Duplicate = 读 `bandParams(row)` 再 `addBand(b, &outIdx)`,完成后把焦点切到 `outIdx`。
4. **信号订阅**:
   - 见 G1 关闭后再决定具体方式 (option A: 直接 connect AudioEQ 新信号;option B: 连 EqualizerModel 内部信号,需要 AudioEQ 提供访问器)。
   - `bandParamsChanged(int idx)` → `refreshRow(idx)` (单行,不重建整表)。
   - `focusedBandChanged(int idx)` → `m_table->selectRow(idx)` (用 `QSignalBlocker` 包裹避免循环)。
5. **嵌入控件回环防护**:`refreshRow` 与 `rebuildTable` 内的 ComboBox / CheckBox 写值前必须 `QSignalBlocker` 包裹,不能依赖"先 setCurrentIndex 不会触发信号"——`QComboBox::setCurrentIndex` 在值变化时**会**发 `currentIndexChanged`。
6. **自测清单**:
   - Bands SpinBox 改值 → 表格行数同步;StatusBar `count=...` 更新。
   - 拖动 AudioEQ Widget 中的 Band Handle → 表格对应行的 Freq/Gain 列实时更新。
   - 点表格行 → RightPanel 切换到该 band。
   - 表格 Bypass CheckBox 勾选 → 曲线对应 band 旁路。
   - 右键 Remove → 表格行数减少;Duplicate → 表格行数增加且新 band 被 focus。

## 4. 已知缺口与待澄清事项 (开发前必须由用户确认)

> 以下两项是本指南撰写过程中识别出的、超出 [问题清单.md](./问题清单.md) 当前条目范围的不确定性。按 [Agent开发规范](../../Agent开发规范.md) 第 1.3 节与第 3.3 节,**必须由 MainAgent 追加到 [问题清单.md](./问题清单.md) 并求得用户确认后再进入对应 SubStage**。

### G1 — `AudioEQ` 未公开暴露 `bandAdded` / `bandRemoved` / `bandCountChanged` 信号

**事实**:[src/AudioEQ.h:74-77](../../../src/AudioEQ.h#L74-L77) 当前 `signals` 段只有:
```cpp
void bandParamsChanged(int index);
void focusedBandChanged(int index);
void sampleRateChanged(SampleRate rate);
```
但 [设计文档.md](./设计文档.md) 第 9C 节"响应信号"明确要求订阅 `bandAdded(int) / bandRemoved(int)`,并以此驱动表格行的增删。这两个信号目前**只**存在于 [src/EqualizerModel.h:68-69](../../../src/EqualizerModel.h#L68-L69),`EqualizerModel*` 是 `AudioEQ` 的私有成员,9C 拿不到。

**影响**:9C 在不解决 G1 之前无法实现"setBandCount → 表格自动同步行数",也无法在 Add Band / Remove Band 时即时更新表格。

**候选方案 (供用户选择)**:
- **A. 给 `AudioEQ` 公共 API 增加两个信号 `bandAdded(int)` / `bandRemoved(int)`**,在 [src/AudioEQ.cpp:205-213](../../../src/AudioEQ.cpp#L205-L213) 的 `onBandAdded` / `onBandRemoved` 末尾 `emit` 即可。改动量极小,语义最干净,但触碰了 SDK 公共表面 ([设计文档.md](./设计文档.md) "非目标:不实现新的 SDK 能力"——*暴露已有内部信号属于扩展接口,不是新能力,边界请用户裁定*)。
- **B. 给 `AudioEQ` 加一个 `EqualizerModel* model() const` 访问器** 或友元 `BandsTablePanel`,让 9C 直接 connect 内部信号。破坏封装,**不推荐**。
- **C. 9C 用 `QTimer` 周期性 `bandCount()` 轮询**。可行但有滞后/抖动,不推荐。

**指南推荐**:**方案 A**。理由:改动局部,语义清晰,与现有 `bandParamsChanged / focusedBandChanged` 风格一致;且现有 `onBandAdded/onBandRemoved` 槽函数已经存在,只需添加两行 `emit`。

**升级路径**:MainAgent 把 G1 追加到 [问题清单.md](./问题清单.md) 的"Stage9 内追加的问题"表格 (作为 `S9-Q7`),用户答复后:
- 若用户选 A → MainAgent 在 9A 范围内**或**作为 Stage9 前置小修做一次 SDK 变更 (修改 AudioEQ.h + AudioEQ.cpp),全量回归 Stage8 验收清单确认无新 regression。
- 若用户选其他方案 → 重新规划 9C 实现,本指南第 3.3 节相应小节重写。

**G1 关闭之前**:9C SubAgent **不开工** (允许先做 BandsCount SpinBox 这一不依赖信号的部分;表格信号订阅暂停)。

### G2 — SubStage 9D (字体 atlas 修复) 设计缺失

**事实**:[问题清单.md](./问题清单.md) S9-Q1 决议把字体 atlas 修复并入 Stage9 作为 SubStage 9D,但 [设计文档.md](./设计文档.md) 没有给 9D 撰写小节 (设计文档"风险与说明 5"反而说"本 Stage 不修这个,由独立任务处理")。两者矛盾。

## 6. 禁止事项 (所有 SubStage 通用)

1. **禁止修改 [src/](../../../src/) 下任何文件**,除非 G1 / G2 解决后用户明确批准 (届时 SDK 改动只由 MainAgent 在 9A 范围内执行)。SubAgent 在任何情况下都不能"顺手"改 SDK。
2. **禁止引入新第三方依赖** ([总览.md](../总览.md) 全局技术决策已锁定)。9A/9B/9C 仅可使用 `Qt6::Widgets` 与已链接的 `AudioEQ` 库。
3. **禁止 SubAgent 之间直接调用对方头文件中的内部方法**。所有跨 Panel 协作必须经过 `AudioEQ` 信号 或 `DemoMainWindow::setStatusMessage`。
4. **禁止使用 `disconnect/connect` 切换或手工 `bool m_updating` flag 防止信号回环**——统一 `QSignalBlocker`。
5. **禁止在 UI 控件中持久缓存 SDK 状态**。每次 SDK 信号到达后从 SDK 重读。
6. **禁止把 `setPointCount` / `setCurveColor` / `setBackgroundColor` / `setFocusedBandIndex` 的返回值"包"进 `apply()`**——它们是 `void`,不返回 `ResultCode`。`apply()` 仅用于 `ResultCode`-returning setter。
7. **禁止在 9A 完工前并行启动 9B/9C** 的代码合入 (设计期可以并行,但代码需要 9A 的 `DemoMainWindow` 与 CMake 集成基础)。
8. **禁止跳过自测清单**直接报告完成——SubAgent 自测是"完成"的必要条件 ([Agent开发规范](../../Agent开发规范.md) 第 3.3 节第 4 步)。
