## 背景与目标

Stage6 交付的 `examples/SimpleDemo/main.cpp` 仅演示了 `setSampleRate / setBandCount / setLpfEnabled / setHpfEnabled` 4 个接口，而 `AudioEQ` 公共接口（[src/AudioEQ.h:25-72](../../../src/AudioEQ.h#L25-L72)）实际暴露了约 30 个方法，覆盖 6 大类能力：

| 分类 | 已演示 | 公共接口数量 |
|------|-------|------|
| Band 管理（增删改查） | ❌ | 9 |
| Band 焦点 | ❌ | 2 |
| 采样率 | ✅（部分） | 2 |
| LPF / HPF | ✅（部分） | 8 |
| Q / Gain 范围与点数 | ❌ | 6 |
| 颜色与外观 | ❌ | 2 |

> **问题**：当前 SimpleDemo 不能验证大部分 SDK 能力，对外部使用者也不能展示 SDK 的真实表面。
>
> **本 Stage 目标**：把 SimpleDemo 改造为**全接口交互面板**——预览窗口（`AudioEQ` Widget）依然是视觉重心，同时在四周布置控件面板，可以在不写一行代码的情况下调用 SDK 的全部公共方法，并实时观察渲染效果。

> **非目标**：
> - 不实现新的 SDK 能力，仅消费现有公共接口
> - 不引入持久化（presets/JSON 导入导出留给后续版本）
> - 不做美工级 UI 设计，控件以"清晰、密度高"为优先

UI 用 `QMainWindow`：
- **CentralWidget** = `AudioEQ` Widget（预览）
- **TopBar** = `QToolBar`（addToolBar(Top)）
- **RightPanel / BottomPanel** = `QDockWidget`（addDockWidget(Right) / Bottom），用户可拖出/收起
- **StatusBar** = `QStatusBar`

> 选用 Dock 而非 Splitter：可缩可隐，方便用户把 AudioEQ Widget 放大到全窗口看渲染细节，同时不丢失控件入口。

## SubStage 划分

### 9A — 顶部控件（采样率 + 全局滤波器）+ 主框架重构

**依赖**：无（可独立先开工）
**文件**：
- `examples/SimpleDemo/main.cpp`（重构入口）
- `examples/SimpleDemo/DemoMainWindow.h/.cpp`（**新建**，承载 UI 逻辑）
- `examples/SimpleDemo/CMakeLists.txt`（添加新源文件）

**职责**：
1. 把 `main.cpp` 拆成 `DemoMainWindow`（继承 `QMainWindow`）+ 极简 `main`。
2. 实现 TopBar（`QToolBar`）：
   - **SampleRate ComboBox**：枚举 4 个 `SampleRate` 值（44.1k / 48k / 96k / **192k**）；`currentIndexChanged` → `eq->setSampleRate(...)`；`AudioEQ::sampleRateChanged` → 反向同步 ComboBox（注意防回环）。
   - **LPF CheckBox**：`toggled(bool)` → `setLpfEnabled`；初始读 `isLpfEnabled()`。
   - **LPF Freq SpinBox**：double，范围 20–96000，单位 Hz；`valueChanged` → `setLpfFrequency`。
   - **LPF Algo Button**（小弹出）：点击弹出菜单，列出 `FilterAlgorithmType` 全部枚举值；选中后 `setLpfAlgorithm`。
   - HPF 三件套同理。
   - **Reset 按钮**：调用一组默认值序列（`setBandCount(5) / setSampleRate(SR_48000) / setLpfEnabled(false) / setHpfEnabled(false) / setGainRange(-12,12) / setPointCount(500) / setCurveColor(0,255,0) / setBackgroundColor(26,26,26)`），用于恢复初始状态。
   - **Bypass All CheckBox**：勾选后遍历所有 band 调 `setBandBypass(i,true)`，反之置 false。
3. 创建 `QStatusBar`，实现 StatusBar 信号订阅（见 9B/9C，9A 只搭骨架并给 setter 函数）。
4. main 函数中：判 `isVulkanSupported()` → 失败则 `QMessageBox::critical` 并退出。

**接口契约**：`DemoMainWindow` 持有 `AudioEQ* m_eq`，向 9B / 9C 暴露：
```cpp
class DemoMainWindow : public QMainWindow {
public:
    explicit DemoMainWindow(QWidget* parent = nullptr);
    AudioEQ* eq() const { return m_eq; }
    void setStatusMessage(const QString& key, const QString& value); // 用于 StatusBar 更新
};
```

**完成判定**：
- TopBar 上 8 个控件可见
- 每个控件操作后调用对应 SDK 方法（用调试器/日志确认调用一次）
- 启动时 SampleRate ComboBox / LPF/HPF CheckBox 显示的是 `AudioEQ` 真实状态
- 切换采样率后 ComboBox 不发生死循环

### 9B — 右侧 Band Inspector + Add Band + Ranges/Render 组

**依赖**：9A 已搭好 `DemoMainWindow` 框架（接口契约层面；代码可并行编辑）
**文件**：
- `examples/SimpleDemo/BandInspectorPanel.h/.cpp`（**新建**，`QDockWidget` 内嵌 `QWidget`）

**职责**：
1. **Focused Band 选择器**（顶部）：
   - `QComboBox`，刷新时按 `bandCount()` 列出 `#0..#N-1` + 一项 `(none, -1)`
   - `currentIndexChanged` → `eq->setFocusedBandIndex(idx)`
   - 监听 `eq` 的 `focusedBandChanged` 信号反向同步
2. **Per-Band 编辑器**（焦点 band 的属性）：
   - **Type ComboBox**：5 个 FilterType
   - **Algo ComboBox**：1 个 FilterAlgorithmType（v1.0.0 只有 Butterworth；用 ComboBox 留扩展）
   - **Freq Slider+SpinBox 联动**：`QSlider`（log 映射，20–22050 Hz） + `QDoubleSpinBox`，互相同步，最终都调 `setBandFrequency`
   - **Gain Slider+SpinBox**：范围跟 `gainMin()/gainMax()`，dB 单位
   - **Q Slider+SpinBox**：范围跟 `qRange(currentType)`
   - **Bypass CheckBox**
   - **Remove Band 按钮**：调用 `eq->removeBand(focusedIndex)`
   - 当 focusedIndex == -1 时整个组禁用
   - 监听 `bandParamsChanged(int)` 信号——若 `idx == focusedIndex`，重新读 `bandParams(idx)` 刷新所有控件（防回环：刷新时阻塞所有信号）
3. **Add Band 表单**（折叠 GroupBox）：
   - 4 个字段：Type ComboBox / Freq SpinBox / Gain SpinBox / Q SpinBox
   - "Add" 按钮 → 构造 `EQBand` 调 `addBand(b, &outIdx)`，成功后把 focusedBand 切换到 `outIdx`
4. **Ranges 折叠组**：
   - 5 行（每种 FilterType）："min Q" / "max Q" SpinBox → `setQRange(type, min, max)`
   - 1 行 Gain："min dB" / "max dB" → `setGainRange(min, max)`
   - 启动时用 `qRange()` / `gainMin()`/`gainMax()` 读取初值
5. **Render 折叠组**：
   - **Point Count** SpinBox（100–2000，步长 50）→ `setPointCount`
   - **Curve Color** 按钮 → `QColorDialog::getColor` → `setCurveColor`
   - **BG Color** 按钮 → `setBackgroundColor`
   - 按钮上显示一个色样方块（`QPushButton::setIcon` + 单色 `QPixmap`），便于看见当前颜色

**信号回环防护规范**（重要）：
- 所有"读 SDK 状态刷新 UI"的代码必须先 `QSignalBlocker` 阻塞控件信号
- 所有 SDK setter 调用前不要清空控件——调用后由信号触发刷新

**完成判定**：
- 焦点带切换后 RightPanel 立即显示该 band 的真实参数
- Slider 拖动时 SpinBox 同步、AudioEQ 渲染同步
- AddBand 成功后表格更新（要等 9C），focused 切到新 band
- 改 GainRange 后 Gain Slider 范围立即跟随变化
- 点 Curve Color 选成红色后预览曲线变红

### StatusBar（散落在 9A/9B/9C，统一规范）

`DemoMainWindow::setStatusMessage(key, value)` 维护一个 `QMap<QString, QLabel*>`，以 key 为标识维护多段 StatusBar 文字。各 SubStage 分别在以下时机调用：

| Key | 触发时机 | 来源 |
|---|---|---|
| `focused` | `focusedBandChanged` | 9B |
| `count` | `bandAdded / bandRemoved` | 9C |
| `sr` | `sampleRateChanged` | 9A |
| `gainRange` | 改 Gain Range 后 | 9B |
| `qRange` | 改 Q Range 后 | 9B |
| `points` | 改 pointCount 后 | 9B |
| `lastResult` | 任意 setter 返回的 ResultCode | 通用工具函数（建议放 9A 头文件） |

`lastResult` 实现建议：在 `DemoMainWindow` 提供 `void apply(ResultCode rc, const QString& op)`，把所有 setter 调用包成 `apply(eq->setBandFrequency(...), "setBandFrequency")`，rc != OK 时显示红色文字。

## 数据流约定

```
用户操作 UI 控件
    │
    ▼
DemoMainWindow / Panel 槽函数
    │  调用 AudioEQ 公共方法，得到 ResultCode
    ▼
AudioEQ 内部 EqualizerModel 更新 → 发出 bandParamsChanged / focusedBandChanged / sampleRateChanged 等信号
    │
    ▼
DemoMainWindow / Panel 监听信号 → 用 QSignalBlocker 阻塞自己的控件信号 → 重读 SDK 状态 → 更新 UI
```

> **黄金法则**：UI 永远以 `AudioEQ` 的状态为唯一真理来源，控件值不持久缓存。每次 SDK 信号到达就重新拉取一遍。

## 风险与说明

1. **预览区缩水**：加入面板后预览窗口空间被挤占。Dock 可拖出/隐藏，初始尺寸 `resize(1280, 800)` 给主区留 ~720x520。用户可通过收起 Dock 把预览放大。
2. **信号回环**：UI ↔ SDK 双向同步是经典坑点。必须**统一**用 `QSignalBlocker`，不要混用 `disconnect/connect` 或手工 flag。
3. **焦点带切到 -1 的边界**：当全部 band 被删完，UI 必须正确禁用 RightPanel，避免空指针/越界。
4. **Q 范围依赖 Type**：Type 切换后必须重新查 `qRange(type)` 调整 Slider 范围；当前 Q 值若超界，调用 `setBandQ` 会被 SDK clamp 还是 reject？需要观察 SDK 实际行为后选择策略（建议先 clamp，再调用 setter）。该细节在 9B 实现时如发现行为不符合预期，写入本 Stage 的问题清单。
5. **字体 atlas 缺失**：当前预览中坐标轴文字不渲染，仅影响视觉，不影响 Stage9 验收。验收清单不要求文字可见。
