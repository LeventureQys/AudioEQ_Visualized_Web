## tests/CMakeLists.txt 结构

```cmake
include(GoogleTest)   # 提供 gtest_discover_tests

function(add_audioeq_test name)
    add_executable(${name} ${ARGN})
    # CurveEngine 测试需要 Qt 事件循环（信号槽），统一链接 Qt6::Core 即可
    target_link_libraries(${name} PRIVATE AudioEQ gtest gtest_main Qt6::Core)
    gtest_discover_tests(${name})
endfunction()

add_audioeq_test(TestButterworthIIR TestButterworthIIR.cpp)
add_audioeq_test(TestCoordinateMapper TestCoordinateMapper.cpp)
add_audioeq_test(TestCurveEngine TestCurveEngine.cpp)
```

> **gtest 目标名说明**：通过 `add_subdirectory(thirdparty/code/googletest)` 引入时，目标名为 `gtest` / `gtest_main`（非 `GTest::gtest`，那是 `find_package` 的别名）。SubAgent 按实际接入方式选择正确名称。

## SubStage 7A：ButterworthIIR 单元测试

### 文件：`tests/TestButterworthIIR.cpp`

### 依赖
- `src/filter/ButterworthIIR.h`（Stage3-3A 产出）
- `src/filter/FilterAlgorithmFactory.h`（Stage3-3A 更新后的版本）

### 必须覆盖的测试用例

```cpp
// 测试 1：Peak filter 在中心频率处的增益等于设定增益
// 参数：freq=1000Hz, gain=+6dB, Q=1.0, sampleRate=44100
// gtest 风格：每个用例为 TEST(SuiteName, CaseName)。下列为必须覆盖的用例及断言要点。

// 测试 1：Peak filter 在中心频率处的增益等于设定增益
// 参数：freq=1000Hz, gain=+6dB, Q=1.0, sampleRate=44100
// EXPECT_NEAR(iir.evaluateAt(1000, 44100), 6.0, 0.1)
TEST(ButterworthIIR, PeakAtCenter);

// 测试 2：Peak filter 在远离中心（1 倍频程外）趋近 0dB
// 参数：freq=1000Hz, gain=+6dB, Q=1.0
// EXPECT_LT(|evaluateAt(4000,44100)|, 1.0)；EXPECT_LT(|evaluateAt(250,44100)|, 1.0)
TEST(ButterworthIIR, PeakAwayFromCenter);

// 测试 3：Peak filter gain=0 时所有频率响应 ≈ 0dB
// EXPECT_NEAR(evaluateAt(任意频率, 44100), 0.0, 0.001)
TEST(ButterworthIIR, PeakZeroGain);

// 测试 4：LowShelf 低频增益
// 参数：freq=200Hz, gain=+6dB, Q=0.707
// EXPECT_NEAR(evaluateAt(10,44100), 6.0, 0.5)；EXPECT_NEAR(evaluateAt(5000,44100), 0.0, 0.5)
TEST(ButterworthIIR, LowShelf);

// 测试 5：HighShelf 高频增益
// 参数：freq=8000Hz, gain=+6dB, Q=0.707
// EXPECT_NEAR(evaluateAt(20000,44100), 6.0, 0.5)；EXPECT_NEAR(evaluateAt(100,44100), 0.0, 0.5)
TEST(ButterworthIIR, HighShelf);

// 测试 6：bypass 标志（ButterworthIIR 无 bypass，通过 EQBand.bypass 在上层控制，此用例跳过）

// 测试 7：FilterAlgorithmFactory 返回有效指针
// 对所有 FilterType × FilterAlgorithmType::ButterworthIIR 组合，EXPECT_NE(create(...), nullptr)
TEST(ButterworthIIR, Factory);

// 测试 8：不同采样率下中心频率增益一致（IIR 系数随采样率重新计算）
// 参数：freq=1000Hz, gain=+6dB, Q=1.0
// sr=44100 和 sr=48000 下，EXPECT_NEAR(evaluateAt(1000, sr), 6.0, 0.2)
TEST(ButterworthIIR, SampleRateIndependence);
```

## SubStage 7B：CoordinateMapper 单元测试

### 文件：`tests/TestCoordinateMapper.cpp`

### 依赖
- `src/CoordinateMapper.h`（Stage1-1C 产出）

### 必须覆盖的测试用例

```cpp
// 测试 1：freqToX 边界值
// viewport: (0,0,1000,600), sampleRate: SR_44100 → freqMax=22050
// EXPECT_NEAR(freqToX(20.0), 0.0, 0.01)；EXPECT_NEAR(freqToX(22050.0), 1000.0, 0.01)
TEST(CoordinateMapper, FreqToBoundary);

// 测试 2：gainToY 边界值
// gainRange: (-48, +48)
// EXPECT_NEAR(gainToY(+48), 0.0, eps)；gainToY(-48)≈600；gainToY(0)≈300
TEST(CoordinateMapper, GainToBoundary);

// 测试 3：往返精度
// EXPECT_NEAR(xToFreq(freqToX(1000)), 1000, 0.01)
// EXPECT_NEAR(yToGain(gainToY(0)), 0, 0.001)；yToGain(gainToY(-12))≈-12
TEST(CoordinateMapper, RoundTrip);

// 测试 4：采样率更新影响 freqMax
// 初始 SR_44100 → freqMax=22050；setSampleRate(SR_48000) → freqMax=24000
// EXPECT_NEAR(freqToX(24000), viewport.right(), 0.01)
TEST(CoordinateMapper, SampleRateUpdate);

// 测试 5：setGainRange 更新后坐标映射变化
// setGainRange(-24, +24) 后 gainToY(+24)≈top, gainToY(-24)≈bottom
TEST(CoordinateMapper, GainRangeUpdate);

// 测试 6：toPixel 组合
// toPixel(1000, 0) 应等于 (freqToX(1000), gainToY(0))
TEST(CoordinateMapper, ToPixel);
```

// gtest + Qt 信号：需要 QCoreApplication 事件循环。
// 推荐用 test fixture（::testing::Test 子类）在 SetUp 中创建 QCoreApplication（若尚未存在），
// 等待信号用 QSignalSpy + spy.wait(timeout_ms)（QSignalSpy 来自 Qt6::Test，
// 故 TestCurveEngine 需额外链接 Qt6::Test；或用 QEventLoop + QTimer 超时自行实现）。

// 测试 1：基础异步计算
// 构造 CurveEngine，发起一次 requestTotalCurve
// 用 QSignalSpy 等待 totalCurveReady 信号（超时 3000ms）
// EXPECT_EQ(points.size(), request.pointCount)（默认 500）
// EXPECT_NEAR(points[0].x, 0.0, eps)；EXPECT_NEAR(points[499].x, 1.0, eps)
TEST_F(CurveEngineTest, BasicAsyncRequest);

// 测试 2：快速连续请求取消
// 连续发出 10 次 requestTotalCurve（不等待）；等待信号完成（超时 5000ms）
// EXPECT_LE(收到的 totalCurveReady 次数, 2)（大部分被取消）
TEST_F(CurveEngineTest, RapidCancelation);

// 测试 3：单 Band 曲线
// 发起 requestBandCurve(request, 0)；等待 bandCurveReady(0, ...) 信号
// 验证 points 大小正确
TEST_F(CurveEngineTest, BandCurveRequest);

// 测试 4：析构时不崩溃（活跃计算中销毁）
// 发起 requestTotalCurve，立即 delete CurveEngine；不崩溃即通过
TEST_F(CurveEngineTest, DestructorSafety);

// 测试 5：setPointCount 生效
// setPointCount(100)，发起请求，EXPECT_EQ(points.size(), 100)
TEST_F(CurveEngineTest, PointCount);
