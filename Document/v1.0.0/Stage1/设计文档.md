## 背景与目标

本 Stage 从零建立项目的可编译骨架，为所有后续 Stage 提供：
1. 可用的 CMake 构建系统（跨 Win/Linux/macOS，能够 configure + build，各模块均编译为空实现）
2. 所有共享的数据结构、枚举定义（头文件）
3. `CoordinateMapper` 的完整实现（纯算法，无 GUI 依赖）
4. 4 组 GLSL Shader 源码 + **预编译好的 `.spv` 字节码内置进仓库**（构建机不需要 GLSL 编译器）
5. thirdparty 依赖接入（volk / vulkan-headers / glm / stb / 字体 / googletest）

完成后，仓库中存在一个可以 `cmake -S . -B build && cmake --build build` 成功构建的空项目，所有模块头文件存在但实现为空桩。

### 关键技术基线（摘自全局决策，本 Stage 直接相关）

- **Vulkan 通过 volk 加载**（Q11）：编译进 `thirdparty/code/volk/volk.c`，所有 Vulkan 代码 include `"volk.h"` 而非 `<vulkan/vulkan.h>`。
- **不依赖系统 Vulkan SDK**（Q11）：vulkan 头文件来自 `thirdparty/code/vulkan-headers/include/`，CMake **不使用** `find_package(Vulkan)`。
- **Shader 预编译为 .spv 内置**（Q13）：Stage1 负责生成并提交 `.spv` 文件，不在构建时调用 glslc。
- **字体嵌入字节数组**（Q16）：CMake 把 ttf 转 C 头文件，不用 Qt 资源系统。
- **GLM 仅渲染层**（Q17）：CoordinateMapper（1C）继续用 Qt 类型，不引入 GLM。
- **gtest 优先 submodule**（Q15）。

## SubStage 1C：CoordinateMapper 完整实现

### 目标
实现 `CoordinateMapper`，提供频率/增益与视口像素坐标的双向映射。这是一个**纯算法类，无 GUI 依赖**。

### 接口定义（完整）

```cpp
// src/CoordinateMapper.h
#pragma once
#include <QRectF>
#include <QPointF>
#include "AudioEQTypes.h"

class CoordinateMapper {
public:
    CoordinateMapper();

    // 设置视口大小（像素）和当前 Nyquist 频率
    void setViewport(const QRectF& rect);
    void setSampleRate(SampleRate rate);   // 用于计算 Nyquist = rate/2

    // 频率(Hz) → X 像素坐标（对数映射）
    // freqMin=20Hz, freqMax=Nyquist
    // 公式：x = left + (log(freq/freqMin) / log(freqMax/freqMin)) * width
    double freqToX(double freqHz) const;

    // X 像素坐标 → 频率(Hz)（freqToX 逆运算）
    double xToFreq(double x) const;

    // 增益(dB) → Y 像素坐标（线性映射，上负下正，0dB 在中央）
    // 公式：y = top + (gainMax - gainDb) / (gainMax - gainMin) * height
    double gainToY(double gainDb) const;

    // Y 像素坐标 → 增益(dB)
    double yToGain(double y) const;

    // 组合：(freq, gain) → 像素坐标
    QPointF toPixel(double freqHz, double gainDb) const;

    // 当前视口和增益范围
    QRectF viewport() const;
    void setGainRange(double minDb, double maxDb);
    double gainMin() const;
    double gainMax() const;

    // 频率范围
    double freqMin() const;   // 固定 20.0 Hz
    double freqMax() const;   // = Nyquist（sampleRate / 2）

private:
    QRectF  m_viewport;
    double  m_gainMin  = -48.0;
    double  m_gainMax  = +48.0;
    double  m_freqMin  = 20.0;
    double  m_freqMax  = 22050.0;  // 默认 44100/2
};
```

### 实现要点

- `freqToX` / `xToFreq`：对数映射，使用 `std::log` 自然对数（`log(freq/freqMin) / log(freqMax/freqMin)` 的比值与底数无关）
- `gainToY` / `yToGain`：线性映射，**Y 轴方向**：增益越大 Y 越小（图像上方 = 高增益）
- `setSampleRate` 更新 `m_freqMax = static_cast<int>(rate) / 2.0`
- 所有方法都是 `const`，无副作用

### 验收标准

- `freqToX(20.0)` 返回 `viewport().left()`（误差 < 0.001）
- `freqToX(Nyquist)` 返回 `viewport().right()`
- `gainToY(m_gainMax)` 返回 `viewport().top()`
- `gainToY(m_gainMin)` 返回 `viewport().bottom()`
- `xToFreq(freqToX(1000.0))` ≈ 1000.0（往返精度误差 < 0.01Hz）
- `yToGain(gainToY(0.0))` ≈ 0.0（误差 < 0.001dB）

## SubStage 1D：GLSL Shader 源码 + 预编译 SPIR-V

### 目标
在 `src/vulkan/shaders/` 下创建 8 个 GLSL 源码文件，**并将它们预编译为 `.spv` 字节码提交进仓库**（Q13）。构建机不需要安装 GLSL 编译器；CMake 直接读取已有的 `.spv`。

SubAgent 在自己的开发环境用 `glslc`（来自任意 Vulkan SDK 安装）编译一次，把产物 `.spv` 提交到 `src/vulkan/shaders/spv/`。

### 各 Shader 的职责和接口

#### Grid Shader（grid.vert / grid.frag）
- **职责**：绘制频率/增益网格线（水平线 + 垂直线）
- **输入**（顶点）：`layout(location=0) in vec2 inPosition;`（NDC 坐标 [-1,1]）
- **Uniform**：`layout(set=0,binding=0) uniform GridUBO { vec4 color; float alpha; };`
- **输出**：纯色线段，颜色为 UBO 中的 color，透明度 alpha

#### Curve Shader（curve.vert / curve.frag）
- **职责**：绘制频响曲线（TriangleStrip 宽线）
- **输入**（顶点）：
  ```glsl
  layout(location=0) in vec2 inPosition;  // NDC
  layout(location=1) in float inDist;      // 到曲线中心线的距离（像素），用于抗锯齿
  ```
- **Uniform**：`layout(set=0,binding=0) uniform CurveUBO { vec4 color; float lineWidth; };`
- **Fragment**：用 `smoothstep(lineWidth*0.5-1.0, lineWidth*0.5, abs(inDist))` 做边缘抗锯齿（距离越大越透明）

#### Fill Shader（fill.vert / fill.frag）
- **职责**：绘制聚焦 Band 时曲线下方的半透明渐变填充区域
- **输入**：`layout(location=0) in vec2 inPosition;` + `layout(location=1) in float inAlpha;`（顶部 alpha 高，底部趋近 0）
- **Uniform**：`layout(set=0,binding=0) uniform FillUBO { vec4 color; };`

#### Glyph Shader（glyph.vert / glyph.frag）
- **职责**：绘制文字（频率/增益标注、LPF/HPF 文字）
- **输入**：
  ```glsl
  layout(location=0) in vec2 inPosition;   // NDC
  layout(location=1) in vec2 inTexCoord;   // 字体图集 UV
  ```
- **Uniform**：`layout(set=0,binding=0) uniform GlyphUBO { vec4 color; };`
- **Sampler**：`layout(set=0,binding=1) uniform sampler2D fontAtlas;`
- **Fragment**：采样字体图集的 R 通道作为 alpha（直接位图字体）

### 预编译与提交

SubAgent 在本地用 `glslc`（任意 Vulkan SDK）执行下列命令，把产物提交进仓库：
```
glslc -O src/vulkan/shaders/grid.vert  -o src/vulkan/shaders/spv/grid.vert.spv
glslc -O src/vulkan/shaders/grid.frag  -o src/vulkan/shaders/spv/grid.frag.spv
# ... 对 curve / fill / glyph 的 .vert 和 .frag 同理，共 8 个 .spv
```

同时提供一个可选的重编脚本 `src/vulkan/shaders/compile.sh`（和 `.bat`），记录上述命令，方便后续改 Shader 时手动重编。**该脚本不接入 CMake 构建流程**（避免构建机依赖 glslc）。

### EmbedSpirv.cmake 约定（供 Stage3-3D 使用）

Stage1 的 1A 在 `cmake/EmbedSpirv.cmake` 中提供一个函数，把 `.spv` 文件转为 C 字节数组，生成 `EmbeddedShaders.h/.cpp` 到 build 目录。Stage1 只需保证该函数存在且能对 1D 提交的 8 个 `.spv` 正确生成头文件；实际在 Pipeline 中加载由 Stage3-3D 完成。

生成的 `EmbeddedShaders.h` 接口（与 Stage3-3D 契约 C3-5 一致）：
```cpp

#include <cstdint>
#include <cstddef>
namespace EmbeddedShaders {
    extern const uint32_t grid_vert[];  extern const size_t grid_vert_size;   // 字节数
    extern const uint32_t grid_frag[];  extern const size_t grid_frag_size;
    // ... 其余 6 组（curve / fill / glyph 各 vert+frag）
}
```

### 验收标准

- 8 个 GLSL 源码文件存在且语法正确
- `src/vulkan/shaders/spv/` 下存在对应 8 个 `.spv` 文件（已提交）
