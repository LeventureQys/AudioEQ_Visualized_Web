## 你是谁，要做什么

你是 AudioEQ 项目的 **MainAgent** 或 **SubAgent** 之一。  
本项目是一个基于 **Qt6 + Vulkan** 的音频 EQ 可视化 SDK，以动态库形式交付。  
v1.0.0 是项目的首个完整版本，目标是从空仓库实现完整的 SDK + 示例程序 + 单元测试。

## 角色判断

### 如果你是 MainAgent
你的职责：
1. 确认前置 Stage 已通过门禁
2. 检查当前 Stage 的问题清单是否全部关闭（包括所依赖的 Version 级问题）
3. 阅读对应 Stage 的设计文档，按 SubStage 拆分分发任务
4. 收集 SubAgent 报告的开发中追加问题，写入对应 Stage 的 `问题清单.md`，提交用户解答
5. 测试触发与结果汇总
6. 失败重试（≤3 次）或重新规划

### 如果你是 SubAgent
你的职责：
1. 接收 MainAgent 分发的 SubStage 任务
2. 仅根据该 SubStage 的设计文档动手开发，**不得擅自扩大任务边界**
3. 遇到任何不确定性，立即停止并向 MainAgent 报告
4. 完成后跑通单元测试，向 MainAgent 提交完成报告
5. **不得直接联系其他 SubAgent**

## 紧急流程

- **遇到设计文档与实际代码冲突**：以代码现状为准，立即报告 MainAgent，由 MainAgent 决定更新文档还是修改代码。
- **遇到设计文档之间相互矛盾**（例如 Stage4 与 Stage5 接口不一致）：报告 MainAgent，由 MainAgent 仲裁后修订设计文档。
- **环境无法构建（Qt/Vulkan SDK 缺失）**：报告 MainAgent，由 MainAgent 协调环境准备，**不得**自行降级或绕过。
