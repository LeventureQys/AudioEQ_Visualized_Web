> **Stage**：Stage3 — 依赖 Stage2 的模块  
> **前置 Stage**：Stage2（必须完整完成）  
> **本 Stage 完成后进入**：Stage4 和 Stage7（Stage7 可与 Stage4 并行）

> **全局技术决策**：遵循 [总览.md](../总览.md#全局技术决策所有-stage-必须遵守)。所有 Vulkan 模块 `#include "volk.h"`（不用 `<vulkan/vulkan.h>`）；Shader 用 Stage1 已提交的预编译 `.spv`（不在构建时跑 glslc）；字体用嵌入字节数组（Q16）。

## 背景与目标

Stage2 完成了所有独立核心模块。Stage3 在此之上实现所有**依赖 Stage2 但彼此不依赖**的模块：
- **3A**：`ButterworthIIR`（具体滤波算法，依赖 2A 的接口）
- **3B**：`CurveEngine`（后台线程曲线计算，依赖 2A+2B+1C）

- **3D**：`VulkanPipeline` + SPIR-V 编译集成（依赖 2C + Stage1 Shader）
- **3E**：`VulkanBufferPool`（依赖 2C）
- **3F**：`VulkanFontAtlas`（依赖 2C）
- **3G**：`LpfHandle` + `HpfHandle`（依赖 2D 的 BandHandle）

## SubStage 3A：ButterworthIIR

### 依赖
- `src/filter/FilterAlgorithm.h`（Stage2-2A 产出）
- `src/AudioEQTypes.h`（Stage1-1B 产出）

### 目标
实现二阶 IIR 巴特沃斯滤波器，支持 Peak / LowShelf / HighShelf / LowPass / HighPass 五种类型，提供频率域幅度响应计算。

### 算法说明（参考 basic_code/CalculateCore.cpp）

旧代码中 `MakeCoeffPeakFilter`、`MakeCoeffLowShelf`、`MakeCoeffHighShelf` 以及频率响应计算公式已有参考实现，路径：`Document/basic_code/CalculateCore.cpp`。  
**注意**：旧代码使用单例模式且依赖 Qt 全局变量 `PublicVar`，新实现必须是无状态的纯函数（通过构造参数传入，`evaluateAt` 方法接受 sampleRate 参数）。

核心频率响应计算公式（适用于 Peak / Shelf）：
```
ω = 2π·f / sampleRate
pha = 4·sin²(ω/2)
numerator   = (b0+b1+b2)² + pha·(b0·b2·pha - b1·(b0+b2) - 4·b0·b2)
denominator = (1+a1+a2)²  + pha·(a2·pha - a1·(1+a2) - 4·a2)
dB = 10·log10(numerator / denominator)
```

LowPass / HighPass 使用 `MakeCoeffLowPass` / `MakeCoeffHighPass`（参考旧代码，或使用标准 Butterworth 公式）。

### 禁止事项
- 不依赖 `PublicVar` 或任何全局状态
- 不持有 `QObject` 亲缘关系，不发射信号

### 验收标准
- Peak filter: `evaluateAt(center_freq, sr)` ≈ `gainDb`（误差 < 0.01dB）
- Peak filter: `evaluateAt(center_freq * 10, sr)` 趋近 0dB（离中心 1 倍频程后增益 < 1dB）
- LowShelf: 低于转折频率增益趋近 `gainDb`，高于转折频率趋近 0dB
- `FilterAlgorithmFactory::create(FilterType::Peak, FilterAlgorithmType::ButterworthIIR, ...)` 返回非 null

## SubStage 3B：CurveEngine

### 依赖
- `src/filter/FilterAlgorithm.h`（2A）
- `src/filter/FilterAlgorithmFactory.h`（2A）
- `src/EqualizerModel.h`（2B，用于快照类型定义）
- `src/CoordinateMapper.h`（1C，用于频点分布公式）
- `src/AudioEQTypes.h`（1B）

### 实现要点

**线程模型**：
- 构造时创建 `QThread`，worker 在该线程运行
- 主线程通过 `QueuedConnection` 发送请求（自动跨线程安全）
- 任务队列：使用 `QMutex` + `QWaitCondition` 实现，每次新请求覆盖旧请求

**取消机制**：
```cpp
std::atomic<bool> m_cancelPending{false};

// 每次计算循环内，在每个频点之间检查：
for (int i = 0; i < pointCount; ++i) {
    if (m_cancelPending.load()) return;  // 直接退出，不 emit 信号
    // ... 计算第 i 个频点
}
```

**频点生成**（对数均匀分布）：
```cpp
// points[i].x = freqMin * (freqMax/freqMin)^(i/(N-1))
// 信号中 QPointF.x 为归一化坐标 [0,1]：x_norm = log(f/freqMin) / log(freqMax/freqMin)
// QPointF.y = 各 Band 在该频率的 dB 叠加
```

**总曲线叠加**：
- 遍历所有未 bypass 的 bands，对每个频点，将所有 band 的 dB 响应相加
- 加上 LPF 和 HPF 的 dB 响应（若 enabled 且未 bypass）

**单 Band 曲线**：
- 只计算指定 `bandIndex` 的单个 band 的频率响应

### 验收标准
- 调用 `requestTotalCurve`，最终在主线程收到 `totalCurveReady` 信号
- 快速连续调用两次，只收到一次信号（第一次被取消）
- 返回的 `QVector<QPointF>` 长度等于 `pointCount`（默认 500）

## SubStage 3D：VulkanPipeline（加载预编译 SPIR-V）

### 依赖
- `src/vulkan/VulkanContext.h/.cpp`（2C）
- `src/vulkan/shaders/spv/*.spv`（Stage1-1D 已提交的预编译字节码）

### SPIR-V 集成策略（Q9/Q13：预编译内置，构建机不需要 glslc）

**统一策略（Debug 与 Release 一致）**：不在构建时调用 glslc。CMake 在构建时把 Stage1 提交的 8 个 `.spv` 文件用 `cmake/EmbedSpirv.cmake` 转为 C 字节数组，生成 `EmbeddedShaders.h/.cpp` 到 build 目录，编译进库。`VulkanPipeline` 通过 `EmbeddedShaders::grid_vert` 等符号取得字节码，`vkCreateShaderModule` 加载。

```cmake
# src/vulkan/CMakeLists.txt 片段
include(EmbedSpirv)   # 来自 cmake/，Stage1 提供

set(SPV_DIR ${CMAKE_CURRENT_SOURCE_DIR}/shaders/spv)
embed_spirv(EMBEDDED_SHADERS_SRC
    ${SPV_DIR}/grid.vert.spv  grid_vert
    ${SPV_DIR}/grid.frag.spv  grid_frag
    ${SPV_DIR}/curve.vert.spv curve_vert
    # ... 其余 5 组
)
# embed_spirv 生成 ${CMAKE_CURRENT_BINARY_DIR}/EmbeddedShaders.h/.cpp
target_sources(AudioEQ PRIVATE ${EMBEDDED_SHADERS_SRC})
target_include_directories(AudioEQ PRIVATE ${CMAKE_CURRENT_BINARY_DIR})
```

`EmbeddedShaders.h` 格式（由 EmbedSpirv.cmake 生成，与 Stage1 约定一致）：
```cpp
#pragma once
#include <cstdint>
#include <cstddef>
namespace EmbeddedShaders {
    extern const uint32_t grid_vert[];  extern const size_t grid_vert_size;  // 字节数
    extern const uint32_t grid_frag[];  extern const size_t grid_frag_size;
    // ... 其余 6 组（curve / fill / glyph 各 vert+frag）
}
```

`loadShaderModule(const uint32_t* spvData, size_t spvSize)` 实现：直接 `VkShaderModuleCreateInfo{ .codeSize = spvSize, .pCode = spvData }`。注意 `spvSize` 是**字节数**，传给 Vulkan 的 `codeSize` 也是字节数。

> 如需修改 Shader：编辑 `.glsl` 源 → 运行 `src/vulkan/shaders/compile.sh` 重新生成 `.spv` → 提交。这是手动流程，不进 CMake。

### Pipeline 顶点输入格式

| Pipeline | location 0 | location 1 |
|---------|-----------|-----------|
| Grid    | vec2 pos (NDC) | — |
| Curve   | vec2 pos (NDC) | float dist（到曲线中心线距离，像素） |
| Fill    | vec2 pos (NDC) | float alpha（渐变 alpha） |
| Glyph   | vec2 pos (NDC) | vec2 uv |

### 验收标准
- `cmake --build build` 后 `EmbeddedShaders.h/.cpp` 自动生成，包含 8 组 SPIR-V 数据，**全程不调用 glslc**
- `VulkanPipeline::create(renderPass)` 成功，所有 4 条 pipeline 的 handle 非 null

### 接口定义

```cpp
// src/LpfHandle.h
#pragma once
#include "BandHandle.h"

// LPF 专用手柄：
// - 椭圆形（宽 40px × 高 24px）
// - 仅允许水平拖拽（增益锁定为 0dB）
// - 显示 "LPF" 文字（由 VulkanRenderer 负责渲染文字，LpfHandle 只负责交互区域）
class LpfHandle : public BandHandle {
    Q_OBJECT
public:
    explicit LpfHandle(QWidget* parent = nullptr);

signals:
    // 覆盖父类信号：只传递 deltaX（deltaY 始终为 0）
    // 注意：此信号名与父类 bandDragged 不同，避免混淆
    void lpfDragged(double deltaX);

protected:
    void mouseMoveEvent(QMouseEvent* event) override;  // 忽略 Y 方向移动
    void paintEvent(QPaintEvent* event)     override;  // 透明，不绘制
};
```

`HpfHandle` 与 `LpfHandle` 完全对称（信号名改为 `hpfDragged`）。

### 验收标准
- 垂直拖拽 `LpfHandle` 时不发出信号（或 `deltaX = 0`）
- 水平拖拽时 `lpfDragged(deltaX)` 信号发出
