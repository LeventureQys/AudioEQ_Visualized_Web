# Stage2 设计文档

> **Stage**：Stage2 — 核心无依赖模块  
> **前置 Stage**：Stage1（需要 `AudioEQTypes.h`、CMake 骨架、volk 已接入）  
> **本 Stage 完成后进入**：Stage3  
> **全局技术决策**：遵循 [总览.md](../总览.md#全局技术决策所有-stage-必须遵守)。Vulkan 用 volk 加载（Q11），Qt 创建 instance、volk 从中加载函数指针（Q12）。  
> **注意**：本 Stage 包含 4 个完全并行的 SubStage，分配给 4 个 SubAgent 同时执行。

---

## 背景与目标

Stage1 建立了可编译的空骨架。Stage2 在此基础上实现所有**不互相依赖**的核心模块：
- **2A**：滤波算法抽象层（接口 + 工厂，不含具体算法实现）
- **2B**：数据模型层（EqualizerModel，采样率/焦点逻辑内嵌，**不建独立 Manager 类**——Q5/Q6）
- **2C**：Vulkan 设备层（VulkanContext + 静态检测，**用 volk 加载**——Q11/Q12）
- **2D**：Band 交互 Widget（BandHandle，纯 Qt，无 Vulkan）

完成后，这 4 个模块可以独立编译和单元测试，Stage3 的所有任务均依赖它们。

### 契约 C2-1：FilterAlgorithm 抽象接口（2A 产出，供 Stage3-3A 使用）

```cpp
// src/filter/FilterAlgorithm.h
#pragma once
#include "../AudioEQTypes.h"

class FilterAlgorithm {
public:
    virtual ~FilterAlgorithm() = default;

    // 计算给定频率的幅度响应（dB）
    // freqHz: 频率（Hz，范围：大于0）
    // sampleRate: 采样率（Hz，如 44100）
    // 返回值：dB，正数为增益，负数为衰减
    virtual double evaluateAt(double freqHz, double sampleRate) const = 0;

    // 子类不可修改，不可持有可变状态（纯函数要求）
};
```

## SubStage 2B：EqualizerModel（采样率/焦点逻辑内嵌）

### 目标
实现数据模型层。存储所有 EQ 参数，通过 Qt 信号通知变更。

### 职责边界说明（Q5/Q6 决策）

**采用内嵌方案，不创建独立的 Manager 类**：
- 采样率管理（`sampleRate()` / `setSampleRate()`）直接作为 `EqualizerModel` 的成员实现
- 焦点管理（`focusedBandIndex()` / `setFocusedBandIndex()`）也直接内嵌在 `EqualizerModel`
- **不创建** `SampleRateManager.h/.cpp` 和 `BandFocusManager.h/.cpp`（旧代码已废弃，从零重做）

### 完整接口

```cpp
// src/EqualizerModel.h
#pragma once
#include <QObject>
#include <QVector>
#include "AudioEQTypes.h"

class EqualizerModel : public QObject {
    Q_OBJECT
public:
    explicit EqualizerModel(QObject* parent = nullptr);

    // ── Band 操作 ──
    int  bandCount() const;
    ResultCode addBand(const EQBand& band, int* outIndex = nullptr);
    ResultCode removeBand(int index);
    EQBand band(int index) const;
    ResultCode setBand(int index, const EQBand& params);
    ResultCode setBandFrequency(int index, double freqHz);
    ResultCode setBandGain(int index, double gainDb);
    ResultCode setBandQ(int index, double q);
    ResultCode setBandType(int index, FilterType type);
    ResultCode setBandAlgorithm(int index, FilterAlgorithmType algo);
    ResultCode setBandBypass(int index, bool bypass);

    // ── LPF ──
    ShelfBand lpf() const;
    ResultCode setLpfEnabled(bool enabled);
    ResultCode setLpfFrequency(double freqHz);
    ResultCode setLpfAlgorithm(FilterAlgorithmType algo);

    // ── HPF ──
    ShelfBand hpf() const;
    ResultCode setHpfEnabled(bool enabled);
    ResultCode setHpfFrequency(double freqHz);
    ResultCode setHpfAlgorithm(FilterAlgorithmType algo);

    // ── 采样率 ──
    SampleRate sampleRate() const;
    ResultCode setSampleRate(SampleRate rate);

    // ── 焦点 ──
    int  focusedBandIndex() const;          // -1 = 无焦点
    void setFocusedBandIndex(int index);    // -1 = 清除焦点

    // ── 增益范围 ──
    double gainMin() const;
    double gainMax() const;
    ResultCode setGainRange(double minDb, double maxDb);

    // ── Q 值范围（每种 FilterType 独立） ──
    ResultCode setQRange(FilterType type, double minQ, double maxQ);
    double qMin(FilterType type) const;
    double qMax(FilterType type) const;

signals:
    void bandChanged(int index);
    void bandAdded(int index);
    void bandRemoved(int index);
    void lpfChanged();
    void hpfChanged();
    void sampleRateChanged(SampleRate rate);
    void focusedBandChanged(int index);
    void gainRangeChanged(double minDb, double maxDb);

private:
    QVector<EQBand>              m_bands;
    ShelfBand                    m_lpf;
    ShelfBand                    m_hpf;
    SampleRate                   m_sampleRate = SampleRate::SR_44100;
    int                          m_focusedIdx = -1;
    double                       m_gainMin    = -48.0;
    double                       m_gainMax    = +48.0;
    QMap<FilterType, QPair<double,double>> m_qRanges;   // FilterType → (min, max)
};
```

### 初始化行为

构造函数中：
1. 初始化 `m_lpf.freqHz = 20000.0; m_lpf.enabled = false;`
2. 初始化 `m_hpf.freqHz = 20.0; m_hpf.enabled = false;`
3. 初始化默认 Q 范围：所有 FilterType 默认 `(0.1, 10.0)`
4. **不**调用 `addBand()`（由 AudioEQ 顶层类在构造时调用 `setBandCount(5)`）

### Q 值 clamp 行为

调用 `setQRange(type, min, max)` 后，必须遍历所有 `m_bands`，对匹配 `type` 的 band 执行 `band.q = clamp(band.q, min, max)`，并为每个被修改的 band 发出 `bandChanged(index)` 信号。

### 验收标准

- `addBand` / `removeBand` / 各 `setBandXxx` 方法均已实现并发出正确信号
- `bandCount()` 初始为 0
- `setGainRange(-24, +24)` 后 `gainMin()` == -24, `gainMax()` == +24
- 索引越界时返回 `ResultCode::IndexOutOfRange`，不 crash

## SubStage 2C：VulkanContext + isVulkanSupported()

### 目标
实现 Vulkan 设备初始化层。**使用 volk 加载 Vulkan 函数指针**（Q11），由 **Qt 创建 VkInstance、volk 从中加载**（Q12）。创建逻辑设备和图形队列。

### 依赖与关键约束

- `thirdparty/code/volk/volk.h`（Stage1 已接入）：**所有 Vulkan 符号通过 volk 使用**
- Qt6 `QVulkanInstance`（通过 `Qt6::Gui` 提供）
- **禁止** `#include <vulkan/vulkan.h>`；统一 `#include "volk.h"`
- **禁止** `find_package(Vulkan)` / 链接 `Vulkan::Vulkan`（不依赖系统 SDK）

### volk + Qt 初始化时序（Q12，强制遵守）

```
1. volkInitialize()                          // 加载全局函数（vkCreateInstance 等）
2. m_qtInstance = new QVulkanInstance();
   m_qtInstance->setApiVersion(QVersionNumber(1,3,0));
   #ifndef NDEBUG
   m_qtInstance->setLayers({ "VK_LAYER_KHRONOS_validation" });
   #endif
   // macOS（Q14）：启用 portability enumeration
   #ifdef Q_OS_MACOS
   m_qtInstance->setFlags(QVulkanInstance::...);  // 见下方 macOS 说明
   #endif
3. m_qtInstance->create()                    // Qt 创建底层 VkInstance
4. volkLoadInstance(m_qtInstance->vkInstance()); // ★ volk 从 Qt 的 instance 加载 instance 级函数
5. 枚举物理设备，选支持图形队列的设备
6. vkCreateDevice(...)，获取图形队列
7. volkLoadDevice(m_device);                 // ★ 加载 device 级函数（更快的调用路径）
```

### macOS portability 说明（Q14）

macOS 仅保证编译通过。在 `initialize()` 中，`#ifdef Q_OS_MACOS` 分支需在 instance 创建时附带 `VK_KHR_portability_enumeration` 扩展和 `VK_INSTANCE_CREATE_ENUMERATE_PORTABILITY_BIT_KHR` flag，创建 device 时启用 `VK_KHR_portability_subset`。由于 Qt 主导 instance 创建，扩展通过 `QVulkanInstance::setExtensions()` 添加。真机验证不纳入本 Version 门禁。

### 接口定义

```cpp
// src/vulkan/VulkanContext.h
#pragma once
#include "volk.h"          // 不要 include <vulkan/vulkan.h>
#include <QVulkanInstance>
#include <vector>

class VulkanContext {
public:
    VulkanContext();
    ~VulkanContext();

    // 初始化 Vulkan 实例 + 设备（不含 Swapchain）
    // 成功返回 true，失败返回 false 并在 qWarning 输出原因
    bool initialize();

    void destroy();
    bool isValid() const;

    // 访问器（initialize() 成功后才有效）
    QVulkanInstance*    qtInstance()      const;   // 供 Qt-Vulkan 窗口集成使用（Stage4）
    VkInstance          vkInstance()      const;
    VkPhysicalDevice    physicalDevice()  const;
    VkDevice            device()          const;
    VkQueue             graphicsQueue()   const;
    uint32_t            graphicsFamily()  const;

    // 静态方法：构造 VulkanContext 前检测 Vulkan 是否可用
    // 内部：volkInitialize() + 创建临时 QVulkanInstance 并 create()
    // 不依赖任何 VulkanContext 实例
    static bool isVulkanSupported();

private:
    QVulkanInstance*  m_qtInstance     = nullptr;
    VkPhysicalDevice  m_physicalDevice = VK_NULL_HANDLE;
    VkDevice          m_device         = VK_NULL_HANDLE;
    VkQueue           m_graphicsQueue  = VK_NULL_HANDLE;
    uint32_t          m_graphicsFamily = 0;
    bool              m_valid          = false;
};
```

### 实现要点

- `isVulkanSupported()`：`volkInitialize()` 成功 → 创建 `QVulkanInstance` 并 `create()`，成功则销毁返回 `true`；任一步失败返回 `false`
- `initialize()`：严格按上方 7 步时序执行
- Debug 构建（`NDEBUG` 未定义）：启用 `VK_LAYER_KHRONOS_validation`
- 失败时所有资源全部清理，`m_valid = false`

### 禁止事项
- 不创建 Swapchain（Stage3-3C 的任务）
- 不创建 RenderPass 或 Pipeline（Stage3-3D 的任务）
- 不直接 include `<vulkan/vulkan.h>`，不链接系统 Vulkan SDK

### 验收标准
- 在有 Vulkan 驱动的机器上，`VulkanContext::isVulkanSupported()` 返回 `true`
- `initialize()` 成功后 `isValid()` 为 `true`，各访问器返回有效句柄
- `volkLoadInstance` / `volkLoadDevice` 调用后 Vulkan 函数指针非空（可调一个简单函数如 `vkGetPhysicalDeviceProperties` 验证）
- `destroy()` 后不崩溃，析构函数调用 `destroy()`

## SubStage 2D：BandHandle

### 目标
实现 `BandHandle`：透明的 `QWidget` 子类，覆盖在 Vulkan 渲染区上方，捕获鼠标事件，支持拖拽（60Hz 节流）和点击。

### 接口定义

```cpp
// src/BandHandle.h
#pragma once
#include <QWidget>
#include <QTimer>
#include "AudioEQTypes.h"

class BandHandle : public QWidget {
    Q_OBJECT
public:
    explicit BandHandle(int bandIndex, QWidget* parent = nullptr);

    // 更新此 Handle 在父 Widget 中的像素位置（圆心）
    // 由 ViewEqualizer 在收到 bandChanged 信号后调用
    void setCenter(QPoint center);
    QPoint center() const;

    int bandIndex() const;

signals:
    void bandDragged(int bandIndex, double deltaX, double deltaY);  // 拖拽增量（像素）
    void bandClicked(int bandIndex);
    void bandReleased(int bandIndex);

protected:
    void mousePressEvent(QMouseEvent* event)   override;
    void mouseMoveEvent(QMouseEvent* event)    override;
    void mouseReleaseEvent(QMouseEvent* event) override;
    void paintEvent(QPaintEvent* event)        override;  // 透明，不绘制

private:
    int     m_bandIndex;
    QPoint  m_center;
    QPoint  m_dragStartPos;
    bool    m_dragging = false;

    QTimer  m_throttleTimer;    // 16ms 节流
    QPoint  m_pendingDelta;     // 节流期间累积的增量
    bool    m_hasPending = false;
};
```

**注意**：`bandDragged` 信号传递的是**像素增量**（`deltaX`, `deltaY`），不是绝对坐标。由 `ViewEqualizer`（Stage5）根据 `CoordinateMapper` 转换为频率/增益变化。

### 节流逻辑

```
鼠标移动 → 计算 deltaX/deltaY：
  若 m_throttleTimer 未启动：
    立即 emit bandDragged（第一帧零延迟）
    启动 m_throttleTimer（16ms 单次）
  否则：
    m_pendingDelta 累积增量
    m_hasPending = true

m_throttleTimer 超时 → 若 m_hasPending：
    emit bandDragged(m_pendingDelta)
    重置 m_hasPending = false
    重启 timer
```

### 圆形 HitTest

`BandHandle` 的有效点击/拖拽区域为以 `m_center` 为圆心、半径 20px 的圆形。`mousePressEvent` 中检查点击位置是否在圆内：

```cpp
if ((event->pos() - m_center).manhattanLength() > 28)  // 28 ≈ 20 * sqrt(2)
    event->ignore();
// 更准确：用 (dx*dx + dy*dy) <= 20*20
```

### Widget 属性要求

```cpp
setAttribute(Qt::WA_TransparentForMouseEvents, false);
setAttribute(Qt::WA_NoSystemBackground, true);
setMouseTracking(false);  // 只在 press 后追踪 move
```

### 验收标准
- 鼠标按下圆形内：`bandClicked` 发出
- 拖拽期间：`bandDragged` 信号每 16ms 最多触发一次（第一次立即）
- 鼠标释放：`bandReleased` 发出
- `paintEvent` 不绘制任何内容（Widget 透明）
