# AudioEQ 数据流设计文档

## 1. 数据流全景图

### 1.1 参数设置流

外部调用方通过 AudioEQ 公开 API 设置 EQ 参数，数据经 EqualizerModel 持久化存储后，通过 Qt 信号通知各下游模块。

```
┌──────────────────────────────────────────────────────────────────────────┐
│                           参数设置流                                      │
│                                                                          │
│  外部调用方                                                              │
│     │                                                                    │
│     │ AudioEQ::setBandParams(index, params)                              │
│     │ AudioEQ::setBandFrequency(index, freqHz)                           │
│     │ AudioEQ::setBandGain(index, gainDb)                                │
│     │ AudioEQ::setBandQ(index, q)                                        │
│     │ AudioEQ::setLpfEnabled(enabled)                                    │
│     │ AudioEQ::setHpfFrequency(freqHz)                                    │
│     │ AudioEQ::setSampleRate(rate)                                        │
│     │ ...                                                                │
│     ▼                                                                    │
│  ┌──────────────────────┐                                                │
│  │      AudioEQ          │                                               │
│  │  (QWidget, 对外入口)   │                                               │
│  └──────────┬───────────┘                                                │
│             │ 调用内部方法, 返回 ResultCode                                │
│             ▼                                                            │
│  ┌──────────────────────┐                                                │
│  │   EqualizerModel     │  ◄──── 数据持久化存储                           │
│  │   (QObject)          │                                                │
│  │                      │                                                │
│  │  QVector<EQBand>     │  频段参数列表                                   │
│  │  ShelfBand m_lpf     │  LPF 独立参数                                   │
│  │  ShelfBand m_hpf     │  HPF 独立参数                                   │
│  │  SampleRate m_rate   │  当前采样率                                     │
│  │  int m_focusedIdx    │  当前焦点 Band 索引                             │
│  │  double m_gainMin    │  增益轴下限                                     │
│  │  double m_gainMax    │  增益轴上限制                                     │
│  └──────────┬───────────┘                                                │
│             │ emit 信号                                                   │
│             │                                                             │
│     ┌───────┼───────────────┬──────────────────┬──────────────┐          │
│     │       │               │                  │              │          │
│     ▼       ▼               ▼                  ▼              ▼          │
│  bandChanged  lpfChanged  hpfChanged  sampleRateChanged  focusedBandChanged│
│  bandAdded    ...                                                         │
│  bandRemoved                                                              │
│     │       │               │                  │              │          │
│     ▼       ▼               ▼                  ▼              ▼          │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │                         下游消费者                                 │    │
│  │                                                                   │    │
│  │  AudioEQ (转发信号给外部)    ViewEqualizer (更新 UI 状态)           │    │
│  │  CurveEngine (触发重算)      BandHandle (更新位置)                 │    │
│  │  VulkanRenderer (标记脏区)   外部监听者 (自定义逻辑)                │    │
│  └──────────────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────────────┘
```

### 1.2 曲线计算流

参数变更后，EqualizerModel 发出信号，AudioEQ 捕获后向 CurveEngine 发起异步计算请求。CurveEngine 在后台工作线程执行计算，完成后通过跨线程信号将结果发回主线程。

```
┌──────────────────────────────────────────────────────────────────────────┐
│                           曲线计算流                                      │
│                                                                          │
│  主线程 (GUI Thread)                        后台工作线程 (Worker Thread)  │
│  ════════════════                          ═══════════════════════════   │
│                                                                          │
│  EqualizerModel                                                          │
│     │ emit bandChanged(index)                                            │
│     ▼                                                                    │
│  AudioEQ (槽)                                                            │
│     │                                                                    │
│     │ 1. 从 EqualizerModel 复制参数快照                                   │
│     │    QVector<EQBand> snapshot = model.allBands();                    │
│     │    ShelfBand lpfSnap = model.lpf();                                │
│     │    ShelfBand hpfSnap = model.hpf();                                │
│     │    SampleRate rateSnap = model.sampleRate();                       │
│     │                                                                    │
│     │ 2. 发起异步计算请求                                                  │
│     │    CurveEngine::requestTotalCurve(modelSnapshot)                   │
│     │    [QueuedConnection 跨线程队列]                                    │
│     │                                                                    │
│     │                            ┌──────────────────────────┐            │
│     │                            │  CurveEngine 工作对象      │            │
│     │   ──── queued signal ────▶ │  (moveToThread 到后台)     │            │
│     │                            │                           │            │
│     │                            │  检查取消标志               │            │
│     │                            │  std::atomic<bool> m_cancel│            │
│     │                            │                           │            │
│     │                            │  1. 生成对数频率点列表       │            │
│     │                            │     freq[i] = fMin*(fMax/ │            │
│     │                            │     fMin)^(i/(N-1))       │            │
│     │                            │                           │            │
│     │                            │  2. 对每个频率点, 累加所有  │            │
│     │                            │     Band 的增益贡献:       │            │
│     │                            │     for each band:        │            │
│     │                            │       totalGain +=        │            │
│     │                            │       m_filterAlgo->      │            │
│     │                            │       evaluateAt(freq,    │            │
│     │                            │       rate, band)         │            │
│     │                            │                           │            │
│     │                            │  3. 叠加 LPF/HPF 增益      │            │
│     │                            │     (if enabled)           │            │
│     │                            │                           │            │
│     │                            │  4. 坐标变换               │            │
│     │                            │     x = freqToX(freq)     │            │
│     │                            │     y = gainToY(gain)     │            │
│     │                            │     clamp y to [min,max]  │            │
│     │                            │                           │            │
│     │                            │  5. 构造 QVector<QPointF>  │            │
│     │                            │     → totalCurveReady()   │            │
│     │                            │                           │            │
│     │                            └───────────┬───────────────┘            │
│     │                                        │                            │
│     │  ◀─── queued signal ──── totalCurveReady(QVector<QPointF>)         │
│     │                                        │                            │
│     ▼                                        │                            │
│  AudioEQ (槽, 回到主线程)                      │                            │
│     │                                                                    │
│     │ 3. 转发曲线数据                                                      │
│     ▼                                                                    │
│  ViewEqualizer                                                           │
│     │                                                                    │
│     ▼                                                                    │
│  VulkanRenderer::updateCurveVBO(points)                                  │
│     │ 更新顶点缓冲区                                                       │
│     ▼                                                                    │
│  requestUpdate() → 下一帧渲染                                             │
└──────────────────────────────────────────────────────────────────────────┘
```

### 1.3 用户交互流（拖拽 Band）

用户通过鼠标拖拽 BandHandle，信号经 AudioEQ 处理，更新 Model，触发曲线重算，最终驱动 Vulkan 重绘。

```
┌──────────────────────────────────────────────────────────────────────────┐
│                    用户交互流 (拖拽 Band)                                  │
│                                                                          │
│  用户鼠标操作                                                             │
│     │                                                                    │
│     │ mousePressEvent(QMouseEvent)                                       │
│     ▼                                                                    │
│  ┌──────────────────┐                                                    │
│  │  BandHandle       │  (透明 QWidget 覆盖层)                              │
│  │                    │                                                    │
│  │  1. 记录拖拽起点    │                                                    │
│  │  2. 启动节流定时器  │  m_dragTimer.start(16)  →  ~60Hz                 │
│  │  3. emit bandPressed(index)                                           │
│  └────────┬─────────┘                                                    │
│           │                                                               │
│     ┌─────┴──────────────┐                                               │
│     │                    │                                               │
│     ▼                    ▼                                               │
│  bandPressed(index)   mouseMoveEvent                                     │
│     │                    │                                               │
│     ▼                    │ m_pendingPosition = event.pos()               │
│  AudioEQ                │                                               │
│     │                    │ if (!m_dragTimer.isActive()) {                │
│     │ setFocusedBandIndex │   m_dragTimer.start(16);                     │
│     │                    │   processDrag();  // 立即处理首帧              │
│     │                    │ }                                             │
│     │                    │                                               │
│     │                    ├─── m_dragTimer timeout ───▶                   │
│     │                    │         │                                     │
│     │                    │         ▼                                     │
│     │                    │    processDrag():                             │
│     │                    │    1. 通过 CoordinateMapper                    │
│     │                    │       将像素坐标转为 (freq, gain)              │
│     │                    │       newFreq = xToFreq(m_pendingPos.x)       │
│     │                    │       newGain = yToGain(m_pendingPos.y)       │
│     │                    │    2. Clamp 到有效范围                         │
│     │                    │    3. emit bandDragged(index, freq, gain)     │
│     │                    │                                               │
│     │                    ▼                                               │
│     │              AudioEQ (槽)                                          │
│     │                    │                                               │
│     │                    │ model.setBandParams(index, newParams)         │
│     │                    │   → emit bandChanged(index)                   │
│     │                    │   → 触发 CurveEngine 异步重算                  │
│     │                    │   → (参见 1.2 曲线计算流)                      │
│     │                    ▼                                               │
│     │              VulkanRenderer 重绘曲线                                │
│     │                                                                    │
│  mouseReleaseEvent                                                       │
│     │                                                                    │
│     ▼                                                                    │
│  BandHandle                                                              │
│     │ m_dragTimer.stop()                                                 │
│     │ 重置拖拽状态                                                        │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

### 1.4 渲染流

VulkanRenderer 从 EqualizerModel 和 CurveEngine 输出中读取数据，生成顶点缓冲，提交 GPU 绘制，呈现到交换链。

```
│  ┌──────────────────────┐                                                │

│  │  paintEvent / timer   │                                                │
│  └──────────┬───────────┘                                                │
│             │ 触发渲染                                                    │
│             ▼                                                            │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │                      VulkanRenderer                               │    │
│  │                                                                   │    │
│  │  每帧执行:                                                         │    │
│  │                                                                   │    │
│  │  ┌─────────────┐                                                  │    │
│  │  │ 1. 获取帧    │  vkAcquireNextImageKHR                           │    │
│  │  │    (Semaphore)│  VulkanFrameSync::acquireFrame()                │    │
│  │  └──────┬──────┘                                                  │    │
│  │         │                                                          │    │
│  │         ▼                                                          │    │
│  │  ┌─────────────┐  从数据源准备 VBO                                  │    │
```

### 1.5 采样率切换流

切换采样率会改变 Nyquist 频率，从而影响 X 轴范围和滤波器的频率响应计算。Band 的频率参数保持不变，但曲线需要重新计算。

```
┌──────────────────────────────────────────────────────────────────────────┐
│                          采样率切换流                                     │
│                                                                          │
│  外部调用                                                                 │
│     │ AudioEQ::setSampleRate(SampleRate::SR_96000)                       │
│     ▼                                                                    │
│  ┌──────────────────────┐                                                │
│  │      AudioEQ          │                                               │
│  └──────────┬───────────┘                                                │
│             │                                                            │
│             │ 1. model.setSampleRate(rate)                               │
│             ▼                                                            │
│  ┌──────────────────────┐                                                │
│  │   EqualizerModel     │                                                │
│  │                      │                                                │
│  │  m_sampleRate = rate │  ← 更新存储                                    │
│  │                      │                                                │
│  │  emit sampleRateChanged(rate)                                         │
│  └──────────┬───────────┘                                                │
│             │                                                            │
│     ┌───────┼──────────────────────────────────────┐                    │
│     │       │                                      │                    │
│     ▼       ▼                                      ▼                    │
│  AudioEQ  CoordinateMapper                  CurveEngine                  │
│  (转发信号) setFreqRange(10, nyquistFreq)     (重算曲线)                   │
│     │       │                                      │                    │
│     │       │ X 轴最大刻度会从                    │                    │
│     │       │   22kHz → 24kHz → 48kHz → 96kHz    │                    │
│     │       │ 随采样率变化                         │                    │
│     │       │                                      │                    │
│     │       ▼                                      │                    │
│     │  触发坐标轴网格重建                           │                    │
│     │  VulkanRenderer::rebuildGridVBO()            │                    │
│     │                                              │                    │
│     ▼                                              ▼                    │
│  emit sampleRateChanged(rate)               沿用已有的曲线计算流           │
│  供外部监听者响应                              (参见 1.2)                  │
│                                              Nyquist 变化               │
│                                              → 滤波器频响变化             │
│                                              → 总曲线点集重新计算          │
│                                                                          │
│  关键约束:                                                               │
│    - Band 频率参数不变 (例如 1000Hz 仍是 1000Hz)                           │
│    - Band 在视图中的 X 位置不变 (因为坐标映射实时使用新 Nyquist)            │
│    - 仅曲线形状和坐标轴刻度发生变化                                         │
│    - LPF/HPF 频率不变, 但滤波效果因 Nyquist 变化而不同                      │
└──────────────────────────────────────────────────────────────────────────┘
```

## 2. 线程模型详解

### 2.1 线程划分

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          线程模型                                        │
│                                                                         │
│  ┌─────────────────────────────────────┐   ┌──────────────────────────┐ │
│  │         主线程 (GUI Thread)          │   │   后台工作线程              │ │
│  │                                     │   │   (Worker Thread)         │ │
│  │  - Qt 事件循环 (QApplication::exec)  │   │                           │ │
│  │  - 所有 QWidget 子类及其事件处理      │   │  - CurveEngine 对象        │ │
│  │    · AudioEQ                        │   │    (通过 moveToThread      │ │
│  │    · ViewEqualizer                  │   │     迁移到此线程)           │ │
│  │    · BandHandle / LpfHandle /       │   │                           │ │
│  │      HpfHandle                      │   │  - 所有曲线计算            │ │
│  │  - Vulkan 渲染                      │   │    · computeTotalCurve()   │ │
│  │    · VulkanRenderer                 │   │    · computeSingleBandCurve│ │
│  │    · VulkanContext / Swapchain /    │   │    · 对数频点生成           │ │
│  │      Pipeline / BufferPool /        │   │    · Catmull-Rom 插值      │ │
│  │      FontAtlas / FrameSync          │   │    · 坐标变换               │ │
│  │  - CoordinateMapper                 │   │                           │ │
│  │                                     │   │  - FilterAlgorithm 调用   │ │
│  │  ┌──────────────────────────────┐   │   │    (共享, 纯函数)           │ │
│  │  │ EqualizerModel               │   │   │                           │ │
```

### 2.3 跨线程通信机制

所有跨线程通信通过 Qt 的 **Queued Connection** 实现：

```
  主线程                          后台工作线程
  ═══════                        ════════════

  AudioEQ / ViewEqualizer         CurveEngine (moveToThread 后)
       │                                │
       │ emit someSignal()              │
       │─── queued ──────────────────▶  │ 槽函数入队到
       │    信号参数被复制到             │ 工作线程的事件队列
       │    QMetaCallEvent              │
       │                                │
       │                                │ 工作线程事件循环
       │                                │ 取出事件, 执行槽
       │                                │
       │                     emit totalCurveReady(points)
       │  ◀─── queued ────────────────  │ 结果打包发回
       │  主线程事件循环取出             │ 主线程
       │  执行槽, 更新 VBO              │
```

**关键实现细节**：

```cpp
// CurveEngine 构造时搬迁到后台线程
CurveEngine::CurveEngine(FilterAlgorithm* algo, QObject* parent)
    : QObject(parent)
    , m_filterAlgo(algo)
{
    // 将 CurveEngine 自身移到后台线程
    // 构造函数中还留在主线程，之后 moveToThread
}

// AudioEQ 初始化
void AudioEQ::initCurveEngine()
{
    m_curveEngine = new CurveEngine(m_filterAlgo, this);
    m_curveEngine->setPointCount(m_curvePointCount);

    // 将 CurveEngine 移到后台线程
    // 此后发往 CurveEngine 的信号自动使用 QueuedConnection
    m_workerThread = new QThread(this);
    m_curveEngine->moveToThread(m_workerThread);

    // 连接信号槽 (自动 QueuedConnection 因为跨线程)
    connect(this, &AudioEQ::requestCurveUpdate,
            m_curveEngine, &CurveEngine::requestTotalCurve);

    connect(m_curveEngine, &CurveEngine::totalCurveReady,
            this, &AudioEQ::onTotalCurveReady);

    m_workerThread->start();
}
```

### 2.4 CurveEngine 取消机制

当新的计算请求到达而旧的计算尚未完成时，通过原子标志位取消旧计算：

```cpp
class CurveEngine : public QObject {
    // ...
    std::atomic<bool> m_cancelPending{false};
};

// 主线程调用 (通过 Queued Signal 到达工作线程)
void CurveEngine::requestTotalCurve(const EqualizerModel& modelSnapshot)
{
    // 1. 设置取消标志, 通知正在进行的计算中止
    m_cancelPending.store(true, std::memory_order_release);

    // 2. 等待旧任务退出 (通过 QMetaObject::invokeMethod 排队新任务)
    //    或直接在下次事件循环中执行新计算
    QMetaObject::invokeMethod(this, [this, snapshot = modelSnapshot]() {
        m_cancelPending.store(false, std::memory_order_release);
        doComputeTotalCurve(snapshot);
    }, Qt::QueuedConnection);
}

// 在工作线程执行
void CurveEngine::doComputeTotalCurve(const EqualizerModel& snapshot)
{
    QVector<QPointF> points;
    points.reserve(m_pointCount);

    for (int i = 0; i < m_pointCount; ++i) {
        // 每个频点计算前检查是否被取消
        if (m_cancelPending.load(std::memory_order_acquire)) {
            return;  // 被取消, 不发出结果信号
        }

        double freq = freqAt(i);
        double gain = 0.0;

        for (const auto& band : snapshot.allBands()) {
            if (band.bypass) continue;
            gain += m_filterAlgo->evaluateAt(freq, rate, band);
        }

        // LPF/HPF 叠加 (if enabled)
        if (snapshot.lpf().enabled) { /* ... */ }
        if (snapshot.hpf().enabled) { /* ... */ }

        double x = m_mapper.freqToX(freq);
        double y = m_mapper.gainToY(gain);
        points.append(QPointF(x, y));
    }

    // 再次检查, 避免发出过期结果
    if (m_cancelPending.load(std::memory_order_acquire)) {
        return;
    }

    emit totalCurveReady(points);
}
```

**取消机制要点**：
- `m_cancelPending` 是 `std::atomic<bool>`，单个原子变量，无锁
- 在工作线程的计算循环中**每隔一个频点**检查取消标志
- 新请求到达 → `store(true)` → 旧计算的循环检测到 → 提前退出 → 不发出 signal
- 如果旧计算恰好在 `store(true)` 和 `emit()` 之间完成，结果仍会被发出（可接受，因为更新的结果会紧随其后覆盖）

### 2.5 FilterAlgorithm 无锁共享

```cpp
// FilterAlgorithm 抽象基类 — 设计为线程安全的纯函数
class FilterAlgorithm {
public:
    virtual ~FilterAlgorithm() = default;

    // 关键设计: const 成员函数, 不修改任何内部状态
    // 所有中间计算都在栈上完成
    virtual double evaluateAt(double freqHz, double sampleRate,
                              const EQBand& band) const = 0;
};

// ButterworthIIR 实现示例
double ButterworthIIR::evaluateAt(double freqHz, double sampleRate,
                                   const EQBand& band) const
{
    // 所有中间变量都在栈上分配
    BiquadCoeff coeff;
    switch (band.type) {
    case FilterType::Peak:
        coeff = makePeakFilter(band.frequency, band.q,
                                band.gain, sampleRate);
        break;
    case FilterType::LowShelf:
        coeff = makeLowShelf(band.frequency, band.q,
                              band.gain, sampleRate);
        break;
    // ... 其他类型
    }
    return freqResponseGain(coeff, freqHz, sampleRate);
}
// 全程无成员变量写入, 无全局/静态变量访问
// 多线程安全, 无需任何锁
```

```
┌─────────────────────────────────────────────────────────────────────────┐
```

## 附录 A：线程安全总结

| 资源 | 访问模式 | 保护机制 |
|---|---|---|
| `EqualizerModel::m_bands` | 主线程写, 主线程读; 后台线程读快照副本 | 快照复制 (值语义) |
| `EqualizerModel::m_lpf / m_hpf` | 主线程写, 主线程读; 后台线程读快照副本 | 快照复制 (值语义) |
| `EqualizerModel::m_sampleRate` | 主线程写, 主线程读; 后台线程读快照副本 | 快照复制 (值语义) |
| `CurveEngine::m_cancelPending` | 主线程写 (通过 Queued Signal), 后台线程读/写 | `std::atomic<bool>` |
| `FilterAlgorithm` 实例 | 后台线程(纯函数调用), 主线程(仅创建) | const 纯函数, 无状态修改 |
| `QVector<QPointF>` 结果 | 后台线程产生, 主线程消费 | QueuedConnection 隐式复制 |
| `Vulkan VBO / GPU 资源` | 主线程独占 | 单线程访问, 无需保护 |
| `QWidget` 子类 | 主线程独占 | Qt 框架保证 (GUI 线程亲和性) |

## 附录 B：数据流关键约束

1. **快照语义**：CurveEngine 接收的参数是 EqualizerModel 的**值拷贝**，后续 Model 的修改不会影响正在进行的计算
2. **取消非阻塞**：`m_cancelPending.store(true)` 立即返回，工作线程在下一个频点迭代时检测并退出
3. **FIFO 结果处理**：虽然后台计算可能被取消多次，但最终到达主线程的 `totalCurveReady` 信号一定是最新一次完整计算的结果（之前的结果被取消而不会发出）
4. **resize 不重算**：窗口 resize 时仅更新 `CoordinateMapper::setViewport()`，曲线数据无需重新计算，仅网格 VBO 需要重建以适应新的像素坐标
5. **默认同步渲染**：非拖拽态使用 `VK_PRESENT_MODE_FIFO`，渲染频率由 VSync 决定，节能且无撕裂
6. **拖拽节流**：拖拽态通过 `QTimer(16ms)` 限制计算和渲染频率 ≤ 60Hz，避免不必要的中间态计算
